## 一、领域概述

### 1.1 领域定义

通用办公技能涵盖各行业、各领域通用的办公场景，包括文档写作、流程审批、信息管理、会议管理、日程管理、沟通协作等基础办公职能。这些技能不依赖专业知识，主要依靠通用AI能力即可完成。

### 1.2 领域特征

| 特征 | 描述 |
|------|------|
| **通用性** | 各行业、各岗位都需要 |
| **标准化** | 有明确的格式和规范 |
| **基础性** | 是其他专业领域的基础 |
| **易实现** | AI能力强项，容易赋能 |
| **高频次** | 使用频率高，价值明显 |
| **低风险** | 错误容忍度高，可人工修正 |

### 1.3 用户群体

| 用户类型 | 典型岗位 | 主要需求 |
|---------|---------|---------|
| **行政人员** | 行政助理、办公室文员 | 文档处理、会议安排、接待工作 |
| **管理人员** | 部门经理、项目主管 | 报告撰写、流程审批、决策支持 |
| **专业人员** | 工程师、设计师、分析师 | 文档写作、数据分析、沟通协作 |
| **销售客服** | 销售代表、客服专员 | 客户沟通、信息整理、报告撰写 |

### 1.4 AI赋能优势

| 优势维度 | 说明 |
|---------|------|
| **效率提升** | 大幅提升办公效率，节省50%-80%时间 |
| **质量保障** | 格式规范、逻辑清晰、表述准确 |
| **成本降低** | 减少人力成本，降低对专业能力的要求 |
| **标准化** | 统一格式和规范，提升整体质量 |

#### 基础信息

```yaml

skill_name: "邮件撰写"
skill_category: "通用办公"
skill_level: "基础"
skill_version: "1.0"

```

#### 技能描述

**简要描述**：根据邮件需求，撰写规范、得体的邮件内容。

**详细说明**：邮件是日常办公的重要沟通工具，需要根据收件人身份、邮件目的、沟通内容等因素，撰写规范、得体、清晰的邮件。

**适用场景**：
- 工作邮件
- 商务邮件
- 通知邮件
- 感谢邮件

#### 知识要求

| 知识类别 | 知识名称 | 掌握程度 | 知识来源 |
|---------|---------|---------|---------|
| **必备知识** | 邮件写作规范 | 熟悉 | 内置 |
| **必备知识** | 商务礼仪知识 | 了解 | 内置 |

#### 能力要求

| 能力类别 | 能力名称 | 能力等级 | 能力描述 |
|---------|---------|---------|---------|
| **核心能力** | 邮件写作能力 | 中等 | 撰写规范、得体的邮件 |
| **核心能力** | 表达能力 | 中等 | 清晰准确地表达信息 |

#### 触发机制

**精确匹配关键词**：
- 写邮件
- 邮件撰写
- 邮件编写
- 起草邮件
- 撰写邮件

**模糊匹配模式**：
- `邮件.*撰写` (置信度: 0.95)
- `邮件.*编写` (置信度: 0.9)
- `邮件.*起草` (置信度: 0.85)

**语义匹配**：
- 核心词：邮件撰写
- 同义词：邮件编写、邮件写作、起草邮件
- 相关词：商务邮件、工作邮件、通知邮件

**上下文触发**：
- 用户身份：职场人员 (权重: 1.0)
- 场景：邮件沟通 (权重: 1.0)
- 任务类型：文档写作 (权重: 0.9)

#### 执行流程

| 步骤 | 步骤名称 | 步骤描述 | 输入 | 输出 | 使用工具 |
|------|---------|---------|------|------|---------|
| 1 | 需求分析 | 分析邮件目的和收件人 | 邮件需求描述 | 邮件需求分析 | - |
| 2 | 内容构思 | 构思邮件结构和内容 | 需求分析结果 | 邮件大纲 | 邮件模板库 |
| 3 | 内容撰写 | 撰写邮件正文 | 邮件大纲 | 邮件初稿 | - |
| 4 | 格式优化 | 优化邮件格式和用语 | 邮件初稿 | 邮件终稿 | 格式检查工具 |

#### 输出标准

**格式标准**：
- ✅ 遵循邮件格式规范
- ✅ 主题清晰，正文简洁
- ✅ 用语得体，表述清晰

**内容标准**：
- ✅ 邮件主题：简洁明确，概括邮件内容
- ✅ 称呼得体：根据收件人身份选择合适称呼
- ✅ 正文清晰：开头说明目的，主体详细阐述，结尾礼貌结束
- ✅ 落款规范：包含签名和联系方式

**质量标准**：
- 准确性：≥ 95% (信息准确率)
- 完整性：≥ 95% (必要信息完整度)
- 规范性：≥ 90% (格式规范符合率)
- 得体性：≥ 90% (用语得体性)

#### 示例

**输入示例**：
```
邮件需求：
  - 收件人：部门经理
  - 邮件目的：请示项目预算调整
  - 主要内容：项目进展、预算超支原因、调整方案
  
用户身份：项目经理
```

**输出示例**：
```
主题：关于XX项目预算调整的请示

王经理：

您好！

现就XX项目预算调整事宜向您请示。

一、项目进展情况
截至本月，XX项目已完成总体进度的75%，各项工作按计划推进。主要完成工作包括：
1. 已完成用户需求调研，形成需求分析报告；
2. 已完成系统架构设计，通过技术评审；
3. 正在进行系统开发工作，预计下月底完成。

二、预算超支原因
经核算，项目预算将超支约15%，主要原因如下：
1. 需求变更导致开发工作量增加；
2. 技术方案调整，增加了第三方服务采购费用；
3. 人员投入增加，加班费用超出预期。

三、预算调整方案
为确保项目顺利完成，建议对项目预算进行如下调整：

2. 增加第三方服务费用5万元；
3. 增加人员费用5万元。

妥否，请批示。

此致
敬礼！

2026年7月15日
联系方式：138xxxx1234

  - 使用技能：gen_doc_email_001
```
