## 一、领域概述

### 1.1 领域定义

通用办公技能涵盖所有行业、所有岗位的基础办公需求，包括文档写作、会议管理、流程审批、信息管理、日程管理等通用性工作场景。

### 1.2 领域特征

| 特征 | 描述 |
|------|------|
| **通用性** | 适用于所有行业、所有岗位 |
| **基础性** | 是其他专业领域的基础 |
| **标准化** | 有规范的格式和流程 |
| **高频性** | 日常高频使用 |
| **易学性** | 学习门槛较低 |

### 1.3 用户群体

| 用户类型 | 典型岗位 | 主要需求 |
|---------|---------|---------|
| **行政人员** | 办公室、行政部 | 文档写作、会议管理、信息报送 |
| **管理人员** | 各级管理者 | 报告撰写、审批流程、决策支持 |
| **专业人员** | 各类专业岗位 | 专业文档、工作报告、项目汇报 |
| **所有员工** | 全员 | 日常办公、沟通协作 |

## 三、文档写作类技能

### 3.1 通知公告写作

#### 技能ID
`office_doc_notice_001`

#### 触发关键词
```
通知、公告、发布通知、写通知、起草通知、会议通知、活动通知、
放假通知、调整通知、紧急通知、重要通知
```

#### 能力要求

**知识要求**：
- 通知公告的基本格式和要素
- 公文写作规范
- 不同类型通知的差异

**技能要求**：
- 根据场景选择合适的通知类型
- 准确表达通知事由、时间、地点、对象等要素
- 语言简洁、准确、规范

#### 输出标准

**基本要素**：
```
【通知标题】
主送单位：
正文：
  一、事由背景
  二、具体安排
  三、相关要求
  四、联系方式
落款：
  发文单位
  发文日期
```

**质量要求**：
- 标题简洁明确
- 事由清晰充分
- 要素完整准确
- 语言规范得体
- 格式符合规范

#### 质量验证

**完整性检查**：
- ✅ 标题包含核心事由
- ✅ 主送单位明确
- ✅ 时间、地点、对象要素完整
- ✅ 落款规范

**规范性检查**：
- ✅ 格式符合公文规范
- ✅ 语言准确无歧义
- ✅ 层次清晰分明

### 3.2 工作总结写作

#### 技能ID
`office_doc_summary_002`

#### 触发关键词
```
工作总结、年度总结、季度总结、月度总结、周总结、个人总结、
部门总结、工作汇报、述职报告、工作回顾
```

#### 能力要求

**知识要求**：
- 工作总结的基本结构
- 数据呈现和分析方法
- 问题归纳和对策建议

**技能要求**：
- 系统梳理工作内容
- 提炼工作亮点和成果
- 分析问题和不足
- 提出改进建议

#### 输出标准

**基本结构**：
```
【工作总结标题】
一、工作概况
  1. 基本情况
  2. 主要指标完成情况
  
二、主要工作及成效
  1. 重点工作一
  2. 重点工作二
  3. 重点工作三
  
三、存在问题及原因分析
  1. 问题一
  2. 问题二
  
四、下一步工作计划
  1. 工作目标
  2. 具体措施
  3. 时间安排
```

**质量要求**：
- 数据真实准确
- 成果具体可量化
- 问题分析深入
- 措施切实可行
- 结构清晰完整
