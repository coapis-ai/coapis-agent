## 🏗️ 技能规格结构

每个技能包含以下核心要素：

### 1. 基础信息

```yaml
skill_id: "gov_policy_draft_001"          # 技能唯一标识
skill_name: "政策文件起草"                  # 技能名称
skill_category: "政府业务"                 # 所属领域
skill_level: "专业"                       # 技能等级（基础/专业/高级）
skill_version: "1.0"                     # 版本号
```

### 2. 能力要求

```yaml
knowledge_requirements:                  # 知识要求
  - 政策法规知识
  - 公文写作规范
  - 行业领域知识

capability_requirements:                 # 能力要求
  - 文档生成能力
  - 格式规范能力
  - 逻辑组织能力
  - 专业术语应用

tool_requirements:                       # 工具要求
  - 知识库检索
  - 模板引擎
  - 格式检查
```

### 3. 触发机制

```yaml
trigger_keywords:                        # 关键词触发
  exact_match:                           # 精确匹配
    - "起草政策"
    - "制定政策文件"
    - "政策初稿"
  fuzzy_match:                           # 模糊匹配
    - "政策.*起草"
    - "文件.*制定"
    - "规定.*编写"
  
trigger_contexts:                        # 上下文触发
  - 用户身份：政府工作人员
  - 场景：政策制定
  - 任务：文档起草
```

### 4. 输出标准

```yaml
output_standards:
  format:                                # 格式标准
    - 遵循公文格式规范
    - 标题层级清晰
    - 段落结构合理
  
  content:                               # 内容标准
    - 专业术语准确
    - 逻辑结构清晰
    - 政策依据充分
  
  quality:                               # 质量标准
    - 准确率：≥ 90%
    - 完整性：≥ 95%
    - 规范性：100%
```

### 5. 验证机制

```yaml
validation:
  auto_check:                            # 自动检查
    - 格式规范检查
    - 术语准确性检查
    - 逻辑完整性检查
  
  human_review:                          # 人工审核
    required: true                       # 是否必须
    reviewer: "政策法规专家"
    check_points:
      - 政策合规性
      - 内容准确性
      - 表述规范性
```

## 🔍 技能分级标准

### 一级技能：基础技能

**特征**：
- 标准化程度高
- 通用性强
- 错误容忍度高
- AI能力强项

**示例**：
- 文档格式化
- 数据整理
- 信息检索
- 基础翻译

**AI能力要求**：低

**人工介入程度**：低

---

### 二级技能：专业技能

**特征**：
- 需要领域知识
- 有一定规范
- 错误容忍度中等
- AI需专业训练

**示例**：
- 政策文件起草
- 合同审核
- 项目报告撰写
- 专业翻译

**AI能力要求**：中-高

**人工介入程度**：中

---

### 三级技能：高级技能

**特征**：
- 需要深度专业知识
- 复杂度高
- 错误容忍度低
- AI需专家系统支持

**示例**：
- 政策影响评估
- 风险决策分析
- 医疗诊断建议
- 法律意见书

**AI能力要求**：高-极高

**人工介入程度**：高

## 📈 技能监控指标

### 监控维度

| 维度 | 指标 | 阈值 | 说明 |
|------|------|------|------|
| **准确性** | 技能匹配准确率 | ≥ 85% | AI识别技能的准确率 |
| **完整性** | 技能覆盖完整度 | ≥ 90% | 技能文档的完整性 |
| **时效性** | 知识更新及时性 | ≤ 7天 | 知识库更新周期 |
| **质量性** | 输出质量达标率 | ≥ 90% | 输出符合质量标准 |

## 🔗 技能调用流程

```
用户输入
    ↓
关键词识别 → 技能匹配
    ↓
上下文分析 → 技能激活
    ↓
能力检查 → 知识库检索
    ↓
技能执行 → 输出生成
    ↓
质量检查 → 人工审核（如需）
    ↓
结果输出
```

## 📝 技能文档维护

### 更新机制

1. **定期更新**：每季度审查一次
2. **需求驱动**：根据用户反馈更新
3. **知识更新**：及时更新专业知识库
4. **版本管理**：保留历史版本

### 版本规范

```
版本号格式：MAJOR.MINOR.PATCH

MAJOR：重大变更（技能结构变化）
MINOR：功能增强（新增技能要素）
PATCH：问题修复（修正错误描述）
```
