# 场景分类正确设计方案

> **创建时间**：2026-07-17
> **核心思想**：通用分类查通用场景，领域分类查领域场景，不混在一起

---

## 一、核心思想

### 1.1 正确的理解

**通用分类维度**：
- 显示通用场景（如会议纪要、公文起草、工作报告）
- 按"办公通用"查询 → 返回会议纪要、公文起草、工作报告

**领域分类维度**：
- 显示领域专属场景（如不动产登记、用地审批）
- 按"自然资源"查询 → 返回自然资源领域的专属场景
- **不包含**通用场景（如会议纪要）

**关键原则**：
- ✅ 按什么分类就查什么分类
- ✅ 通用场景只在通用分类下显示
- ✅ 专属场景只在领域分类下显示
- ✅ 不要混在一起

---

### 1.2 错误的设计

**之前的错误**：
```json
{
  "id": "meeting-minutes",
  "category": ["office-common", "natural-resources", "ecological-environment", ...]
}
```

**问题**：
- ❌ 会议纪要同时属于"办公通用"和"自然资源"
- ❌ 按"自然资源"查询会返回会议纪要（错误！）
- ❌ 通用场景在领域分类下重复显示

---

### 1.3 正确的设计

**方案A：使用对象结构**：
```json
{
  "id": "meeting-minutes",
  "name": "会议纪要",
  "is_generic": true,
  "category": {
    "nature": "office-common",    // 通用分类
    "domains": []                  // 领域分类（空，因为是通用场景）
  }
}

{
  "id": "land-approval",
  "name": "用地审批",
  "is_generic": false,
  "category": {
    "nature": null,                // 通用分类（null，因为是专属场景）
    "domains": ["natural-resources"]  // 领域分类
  }
}
```

**方案B：使用数组，但区分类型**：
```json
{
  "id": "meeting-minutes",
  "name": "会议纪要",
  "is_generic": true,
  "nature_category": "office-common",     // 通用分类
  "domain_categories": []                  // 领域分类（空）
}

{
  "id": "land-approval",
  "name": "用地审批",
  "is_generic": false,
  "nature_category": null,                // 通用分类（null）
  "domain_categories": ["natural-resources"]  // 领域分类
}
```

## 二、查询逻辑

### 2.1 按通用分类查询

```python
def get_scenes_by_nature(nature_id: str):
    """按通用分类查询场景"""
    all_scenes = load_scenes()
    
    # 只返回属于该通用分类的场景
    filtered = [
        s for s in all_scenes 
        if s['category']['nature'] == nature_id
    ]
    
    return filtered
```

**示例**：
```python
# 查询"办公通用"下的场景
get_scenes_by_nature("office-common")
# 返回：会议纪要、公文起草、工作报告
```

---

### 2.2 按领域分类查询

```python
def get_scenes_by_domain(domain_id: str):
    """按领域分类查询场景"""
    all_scenes = load_scenes()
    
    # 只返回属于该领域的专属场景
    filtered = [
        s for s in all_scenes 
        if domain_id in s['category']['domains']
    ]
    
    return filtered
```

**示例**：
```python
# 查询"自然资源"下的场景
get_scenes_by_domain("natural-resources")
# 返回：用地审批、不动产登记、矿山监管（专属场景）
# 不返回：会议纪要、公文起草（通用场景）
```

## 七、总结

### 7.1 核心改进

| 对比项 | 错误方案 | 正确方案 |
|-------|---------|---------|
| **数据结构** | category是数组，混合通用和领域 | category是对象，分开通用和领域 |
| **查询逻辑** | 按ID过滤，不加区分 | 按维度过滤，严格区分 |
| **统计结果** | 通用11个，领域67个（重复） | 通用11个，领域26个（不重复）|
| **用户体验** | 通用场景在领域下重复显示 | 通用和领域各自独立 |

---

### 7.2 关键原则

1. ✅ **按什么分类就查什么分类**
2. ✅ **通用场景只在通用分类下显示**
3. ✅ **专属场景只在领域分类下显示**
4. ✅ **不要混在一起**
