# 场景分类数据统计说明

> **创建时间**：2026-07-17
> **问题**：不同维度分类后的数量不一致

---

## 一、问题说明

### 1.1 数据统计

| 维度 | 分类数 | 场景总计 |
|------|--------|---------|
| **通用分类** | 7个 | 11个 |
| **领域分类** | 10个 | 67个 |
| **实际场景总数** | - | 11个 |

### 1.2 为什么不一致？

**原因**：场景支持多分类

**通用分类**：
- 每个场景只属于一个通用分类
- 例如：会议纪要只属于"办公通用"
- 总计 = 场景总数 = 11个 ✅

**领域分类**：
- 通用场景在多个领域重复显示
- 例如：会议纪要属于10个领域
- 总计 = 67个（场景被重复计数）❓

---

## 二、这是设计问题还是符合预期？

### 方案A：保持现状（通用场景在多个领域显示）

**优点**：
- ✅ 符合实际使用场景（会议纪要确实是所有领域都需要的）
- ✅ 用户在"自然资源"领域下能看到所有相关场景（包括通用场景）

**缺点**：
- ❌ 统计数字不直观（67 > 11）
- ❌ 可能误导用户

**适用场景**：
- 用户通常只在一个领域下工作
- 希望在当前领域下看到所有相关场景

---

### 方案B：调整统计方式（只统计专属场景）

**通用分类**：
- 办公通用：会议纪要、公文起草、工作报告（3个）
- 审批服务：项目审批（1个）
- ...

**领域分类**：
- 自然资源：专属场景（0个）+ 通用场景（8个）= 8个
- 生态环境：专属场景（0个）+ 通用场景（9个）= 9个
- ...

**问题**：
- 当前所有场景都是通用场景，没有专属场景
- 如果只统计专属场景，领域分类下都是0个

---

### 方案C：分开统计（通用场景 + 专属场景）

**领域分类展示**：
```
自然资源 (8个场景)
├── 通用场景 (8个)
│   ├── 会议纪要
│   ├── 公文起草
│   ├── 工作报告
│   ├── 项目审批
│   ├── 规划编制
│   ├── 执法检查
│   ├── 投诉处理
│   └── 监测分析
└── 专属场景 (0个)
    └── （待补充）
```

**优点**：
- ✅ 统计清晰
- ✅ 用户知道哪些是通用场景，哪些是专属场景

---

### 方案D：调整场景分类逻辑

**问题根源**：场景的category字段同时包含通用分类和领域分类

**调整方案**：
```json
{
  "id": "meeting-minutes",
  "category": {
    "nature": "office-common",                    // 通用分类（单一）
    "domains": ["natural-resources", ...]          // 领域分类（多个）
  }
}
```

**优点**：
- ✅ 分类更清晰
- ✅ 便于分别统计

**缺点**：
- ❌ 需要调整数据结构

## 五、数据结构建议

**调整后的数据结构**：

```json
{
  "id": "meeting-minutes",
  "name": "会议纪要",
  
  "category": {
    "nature": "office-common",                    // 通用分类（单一）
    "domains": [                                  // 领域分类（多个）
      "natural-resources",
      "ecological-environment",
      ...
    ]
  },
  
  "properties": {
    "is_generic": true,                           // 是否通用场景
    "frequency": "high"
  }
}
```

**优点**：
- ✅ 分类清晰，不混淆
- ✅ 便于分别统计
- ✅ 便于前端过滤
