## 二、配置文件说明

### 2.1 scenes.json - 场景配置

**数据结构**：
```json
{
  "id": "meeting-minutes",
  "name": "会议纪要",
  "description": "会议录音转写和纪要生成...",
  "agent_id": "scene-meeting-minutes",
  "icon": "📝",
  
  "properties": {
    "nature": "office-common",           // 性质分类ID
    "nature_name": "办公通用",             // 性质分类名称
    "domains": ["all"],                   // 支持的领域（all=所有领域）
    "domain_names": [...],                // 领域名称列表
    "frequency": "high",                  // 使用频率
    "is_generic": true,                   // 是否通用场景
    "priority": "high"                    // 优先级
  },
  
  "skills": ["audio-transcription", "docx"],
  "welcome_message": "...",
  "tags": {...}
}
```

**关键字段**：
- `properties.nature`: 场景的性质分类（每个场景只有一个）
- `properties.domains`: 支持的领域列表（可以是多个，或`["all"]`表示所有领域）
- `properties.is_generic`: 是否为通用场景（true=支持多领域，false=专属场景）

### 2.2 categories.json - 分类配置

**数据结构**：
```json
{
  "dimensions": {
    "nature": {
      "name": "按性质分类",
      "categories": [
        {
          "id": "office-common",
          "name": "办公通用",
          "icon": "📄",
          "description": "...",
          "sort_order": 1
        }
      ]
    },
    
    "domain": {
      "name": "按领域分类",
      "categories": [
        {
          "id": "natural-resources",
          "name": "自然资源",
          "icon": "🏞️",
          "description": "...",
          "sort_order": 1,
          "context": {
            "keywords": [...],
            "regulations": [...]
          }
        }
      ]
    }
  }
}
```

**关键字段**：
- `dimensions.nature`: 性质分类维度
- `dimensions.domain`: 领域分类维度
- `context`: 领域上下文（用于智能体的领域适配）

### 2.4 view_domain.json - 领域分类视图

**用途**：前端展示"按领域分类"的场景列表

**数据结构**：
```json
{
  "dimension": "domain",
  "dimension_name": "按领域分类",
  "categories": [
    {
      "id": "natural-resources",
      "name": "自然资源",
      "icon": "🏞️",
      "scene_count": 8,
      "context": {
        "keywords": [...],
        "regulations": [...]
      },
      "scenes": [
        {
          "id": "meeting-minutes",
          "name": "会议纪要",
          "is_generic": true,
          "is_domain_specific": false
        }
      ]
    }
  ]
}
```

**关键字段**：
- `is_domain_specific`: 是否为该领域专属场景（false=通用场景，true=专属场景）
- `domain_context`: 专属场景的领域上下文（通用场景为null）

### 3.1 后端API实现（1天）

#### API列表

| API | 方法 | 说明 |
|-----|------|------|
| `/api/scenes` | GET | 获取场景列表（支持维度参数） |
| `/api/scenes/{scene_id}` | GET | 获取场景详情 |
| `/api/scenes/categories` | GET | 获取分类列表（支持维度参数） |
| `/api/scenes/search` | GET | 搜索场景（支持标签筛选） |

#### API示例

**获取按性质分类的场景列表**：
```bash
GET /api/scenes?dimension=nature

Response:
{
  "dimension": "nature",
  "dimension_name": "按性质分类",
  "categories": [...]
}
```

**获取按领域分类的场景列表**：
```bash
GET /api/scenes?dimension=domain

Response:
{
  "dimension": "domain",
  "dimension_name": "按领域分类",
  "categories": [...]
}
```

**获取特定分类下的场景**：
```bash
GET /api/scenes?dimension=nature&category=office-common

Response:
{
  "id": "office-common",
  "name": "办公通用",
  "scenes": [...]
}
```

## 六、总结

### 6.1 核心成果

1. ✅ **数据结构设计完成**：场景配置、分类配置、视图生成
2. ✅ **多维度分类实现**：支持"按性质"、"按领域"两个维度
3. ✅ **扁平展示设计**：每个维度下都是一级分类 + 场景卡片
4. ✅ **通用场景复用**：一个场景可以在多个领域展示

### 6.2 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 分类方式 | 多维度分类 | 用户可以从不同视角查找场景 |
| 展示方式 | 维度切换 + 扁平展示 | 简单直观，符合用户认知 |
| 一级分类数量 | 每个维度不超过10个 | 符合认知经济原则 |
| 场景归属 | 一个场景可以从属于多个分类 | 提高场景复用率 |
