# 政府行业领域场景数据

> **数据目录**：`data/industries/`
> **创建日期**：2026-07-17
> **目标**：为每个政府行业领域建立完整的场景数据和智能体配置

## 一、领域分类

| `forestry-grassland` | 林草湿荒 | 林草 | 森林、草原、湿地、荒漠管理 |
| `ecological-environment` | 生态环境保护 | 生态环保 | 环境监测、污染治理、生态保护 |
| `agriculture-rural` | 农业农村 | 农农 | 农业发展、乡村振兴、农村治理 |
| `development-reform` | 发改 | 发改 | 宏观经济、发展规划、项目管理 |
| `culture-tourism` | 文化与旅游 | 文旅 | 文化遗产、旅游发展、市场监管 |
| `housing-construction` | 住建 | 住建 | 城乡建设、房地产市场、建筑管理 |
| `education` | 教育 | 教育 | 教育管理、学校建设、师资培养 |
| `health` | 卫生健康 | 卫健 | 医疗卫生、公共卫生、健康促进 |

---

### 1.2 场景类型

每个领域包含三类场景：

| 场景类型 | 类型代码 | 说明 | 示例 |
|---------|---------|------|------|
| **办公通用** | `office` | 跨领域通用的办公场景 | 会议纪要、公文起草、工作报告 |
| **领域业务** | `business` | 特定领域的专业业务场景 | 项目审批、规划编制、监测分析 |
| **监管巡查** | `supervision` | 日常监管和现场巡查场景 | 执法检查、隐患排查、应急响应 |

## 三、数据收集方法论

### 3.1 信息来源

1. **官方文件**：
   - 政府网站职责介绍
   - 三定方案（定职能、定机构、定编制）
   - 法律法规文件

2. **业务流程**：
   - 行政审批流程
   - 执法检查流程
   - 日常办公流程

3. **专业知识**：
   - 行业标准规范
   - 技术规程指南
   - 典型案例分析

---

### 3.2 数据提取框架

**对于每个领域，提取以下信息**：

#### 3.2.1 组织架构
- 内设机构设置
- 下属单位情况
- 职能分工

#### 3.2.2 核心职责
- 主要职责清单
- 权力清单
- 责任清单

#### 3.2.3 业务场景
- 办公场景（会议、公文、报告）
- 业务场景（审批、监管、服务）
- 执法场景（检查、处罚、应急）

#### 3.2.4 专业知识
- 法律法规
- 技术标准
- 业务流程
- 专业术语

## 四、场景设计原则

### 4.1 办公通用场景

**特点**：
- 跨领域通用
- 标准化程度高
- 使用频率高

**设计要点**：
- 系统提示词：加入领域专业术语
- 知识要求：领域法规政策
- 技能关联：文档处理技能

**示例**：
- 会议纪要（加入领域专业表述）
- 公文起草（符合领域公文格式）
- 工作报告（体现领域工作特点）

---

### 4.2 领域业务场景

**特点**：
- 专业性强
- 流程复杂
- 标准规范严格

**设计要点**：
- 系统提示词：详细的业务流程和专业规范
- 知识要求：行业标准、技术规程
- 技能关联：数据分析、方案设计技能

**示例**：
- 规划编制（国土空间规划）
- 项目审批（用地审批）
- 监测分析（环境监测数据分析）

---

### 4.3 监管巡查场景

**特点**：
- 现场性强
- 执法依据明确
- 处置流程规范

**设计要点**：
- 系统提示词：执法依据、处置流程
- 知识要求：执法标准、典型案例
- 技能关联：信息查询、报告生成技能

**示例**：
- 现场巡查（用地巡查、环保检查）
- 隐患排查（安全隐患、环境隐患）
- 应急响应（突发事件处置）

## 五、实施计划

### 5.1 数据收集阶段（按领域）

**Phase 1：优先领域（3个）**
- 自然资源和规划
- 生态环境保护
- 农业农村

**Phase 2：重要领域（3个）**
- 发改
- 住建
- 教育

**Phase 3：其他领域（4个）**
- 林草湿荒
- 文化与旅游
- 卫健
- 综合执法

---

### 5.2 场景设计阶段

**每个领域包含**：
- 办公场景：3-5个
- 业务场景：5-8个
- 监管场景：3-5个
- **总计**：每个领域约 11-18 个场景

**预期成果**：
- 10个领域 × 15个场景 ≈ **150个场景**

---

### 5.3 智能体初始化阶段

**自动化流程**：
```
1. 读取领域场景数据（scenes.json）
2. 生成场景智能体配置（agent.json）
3. 创建智能体存储目录
4. 初始化共享记忆（MEMORY.md）
```

## 六、使用指南

### 6.1 如何使用这些数据

**开发者**：
1. 查看领域职责（`responsibilities.md`）
2. 了解专业知识（`knowledge-base.md`）
3. 参考场景设计（`scenes.json`）
4. 初始化场景智能体

**业务人员**：
1. 确认本领域场景覆盖情况
2. 提出场景优化建议
3. 补充专业知识内容

---

### 6.2 数据更新机制

**定期更新**：
- 法律法规变化时更新
- 业务流程调整时更新
- 新增场景需求时补充

**版本管理**：
- Git 提交记录变更历史
- 文档版本号标注
- 变更日志记录
