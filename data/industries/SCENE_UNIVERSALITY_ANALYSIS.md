# 场景智能体通用性深度分析

> **分析日期**：2026-07-17
> **分析目的**：识别场景智能体的通用性，避免重复创建，提高复用率

---

## 一、问题识别

### 1.1 当前设计的问题

**当前方案**：10个领域 × 10个场景 = 100个智能体

**问题**：
- 每个领域都有"会议纪要"、"公文起草"、"工作报告"
- 实际上这30个办公场景高度相似
- 区别仅在系统提示词中的专业术语部分

**示例对比**：

| 领域 | 场景名称 | 系统提示词差异 | 核心功能差异 |
|------|---------|--------------|------------|
| 资规 | 会议纪要 | 涉及国土空间规划、土地管理等术语 | 无 |
| 环保 | 会议纪要 | 涉及环评审批、污染防治等术语 | 无 |
| 农业 | 会议纪要 | 涉及乡村振兴、农业发展等术语 | 无 |

**结论**：这30个办公场景可以合并为 **3个通用场景**！

## 二、通用性分析

### 2.1 办公通用场景（30个 → 3个）

#### 场景1：会议纪要（通用）

**原始设计**：10个领域各1个 = 10个智能体
**优化方案**：**1个通用智能体** + 领域参数注入

**通用性分析**：

| 维度 | 通用部分（70%） | 领域特定部分（30%） |
|------|----------------|-------------------|
| **格式要求** | ✅ 时间地点、参会人员、议题、决议、待办事项 | - |
| **核心功能** | ✅ 会议录音转写、纪要生成、待办提取 | - |
| **文档结构** | ✅ 标题、正文、附件格式 | - |
| **专业术语** | - | ⚠️ "三区三线"、"环评"、"乡村振兴"等 |
| **业务流程** | - | ⚠️ 审批流程、执法流程等 |

**优化方案**：

```python
# 通用系统提示词（基础）
system_prompt_base = """
你是会议纪要助手，具备以下能力：
1. 会议录音转写
2. 结构化会议纪要生成
3. 待办事项提取
4. 决议跟踪

会议纪要要求：
1. 格式规范：会议时间、地点、参会人、议题、决议、待办事项
2. 决议清晰：明确责任部门、完成时限
3. 待办事项表格化：责任人、截止时间、完成标准
"""

# 领域参数注入（动态）
domain_context = {
    "natural-resources": "国土空间规划、土地管理、矿产资源等业务",
    "ecological-environment": "环评审批、污染防治、环境执法等业务",
    "agriculture-rural": "农业发展、乡村振兴、农村治理等业务"
}

# 最终系统提示词
system_prompt = system_prompt_base + f"\n\n当前会议涉及：{domain_context[domain]}"
```

**结论**：**可以通用，合并为1个智能体**

### 2.2 领域业务场景（40个 → 25-30个）

#### 分析原则

**判断标准**：
- ✅ **跨领域通用**：多个领域都有类似场景 → 合并
- ⚠️ **领域独特**：仅某个领域有 → 独立
- 🔄 **可参数化**：核心逻辑相同，参数不同 → 参数化

## 四、参数化设计方案

### 4.1 参数注入机制

```python
class SceneAgentConfig:
    """场景智能体配置"""
    
    def __init__(self, base_scene_id: str, domain_code: str = None):
        self.base_scene_id = base_scene_id  # 基础场景ID
        self.domain_code = domain_code       # 领域代码（可选）
        
        # 加载基础场景配置
        self.base_config = self.load_base_config(base_scene_id)
        
        # 如果指定了领域，注入领域参数
        if domain_code:
            self.domain_context = self.load_domain_context(domain_code)
            self.system_prompt = self.inject_domain_context(
                self.base_config['system_prompt'], 
                self.domain_context
            )
        else:
            self.system_prompt = self.base_config['system_prompt']
    
    def inject_domain_context(self, base_prompt: str, domain_context: dict) -> str:
        """注入领域上下文"""
        return f"{base_prompt}\n\n{domain_context['business_context']}"
    
    def load_domain_context(self, domain_code: str) -> dict:
        """加载领域上下文"""
        contexts = {
            "natural-resources": {
                "business_context": "当前业务涉及：国土空间规划、土地管理、矿产资源等",
                "keywords": ["三区三线", "占补平衡", "国土空间规划"],
                "regulations": ["土地管理法", "城乡规划法"]
            },
            "ecological-environment": {
                "business_context": "当前业务涉及：环评审批、污染防治、环境执法等",
                "keywords": ["环评", "排污许可", "空气质量指数"],
                "regulations": ["环境保护法", "大气污染防治法"]
            },
            # ... 其他领域
        }
        return contexts.get(domain_code, {})
```
