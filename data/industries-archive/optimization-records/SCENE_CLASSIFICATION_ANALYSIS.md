# 场景分类体系深度分析与优化方案

> **分析日期**：2026-07-17
> **关键洞察**：场景分类应该是层级化的，当前设计缺少抽象层

---

## 一、问题诊断

### 1.1 当前设计的问题

**当前方案**：
```
场景（一级）：
├── 会议纪要
├── 公文起草
├── 工作报告
├── 项目审批
├── 规划编制
├── 执法检查
└── ...（11个场景）
```

**问题分析**：

| 问题 | 具体表现 | 影响 |
|------|---------|------|
| **分类层级缺失** | 场景直接作为一级分类 | 用户选择困难，认知负担重 |
| **分类逻辑不清晰** | 场景混合了功能和业务 | 分类标准不统一 |
| **扩展性差** | 每个新场景都是一级 | 未来场景增加，一级分类会爆炸 |
| **不符合用户认知** | 用户习惯按功能或领域分类 | 找场景困难 |

**示例对比**：

```
❌ 当前方案（一级分类太多）：
场景列表：
├── 会议纪要
├── 公文起草
├── 工作报告
├── 项目审批
├── 规划编制
├── 执法检查
├── 投诉处理
├── 安全检查
├── 整改验收
├── 应急响应
└── 监测分析
（11个一级分类，用户选择困难）

✅ 优化方案（层级分类）：
功能分类（一级）：
├── 文档处理
│   ├── 会议纪要
│   ├── 公文起草
│   └── 工作报告
├── 审批服务
│   ├── 项目审批
│   └── 规划编制
└── 监管执法
    ├── 执法检查
    ├── 投诉处理
    └── 安全检查
（3个一级分类，层次清晰）
```

## 二、场景分类的本质分析

### 2.1 什么是场景？

**场景定义**：
场景是用户完成某个工作任务的具体情境，包含：
- **任务目标**：用户要做什么（如生成会议纪要）
- **业务流程**：如何完成（如转写→生成→提取待办）
- **专业知识**：需要的专业背景（如资规局的"三区三线"）
- **工具支持**：需要什么技能（如音频转写、文档生成）

**场景特征**：
1. ✅ **粒度适中**：不是泛泛的"办公"，也不是具体的"张三的会议"
2. ✅ **可复用**：不同用户、不同领域可以复用
3. ✅ **可配置**：通过参数适应不同领域
4. ✅ **可进化**：随着使用不断优化

---

### 2.2 场景分类的多维性

**场景可以从多个维度分类**：

#### 维度1：功能属性

| 功能类型 | 典型场景 | 特征 |
|---------|---------|------|
| **文档处理** | 会议纪要、公文起草、工作报告 | 输出文档，格式规范 |
| **审批服务** | 项目审批、用地审批、环评审批 | 流程导向，合规性要求 |
| **规划编制** | 空间规划、发展规划、专项规划 | 方案导向，专业技术性强 |
| **监管执法** | 执法检查、安全检查、投诉处理 | 监督导向，处置流程规范 |
| **数据分析** | 经济分析、监测分析、统计报告 | 数据导向，可视化展示 |

---

#### 维度2：业务领域

| 业务领域 | 典型场景 | 专业特征 |
|---------|---------|---------|
| **自然资源管理** | 用地审批、不动产登记、矿山监管 | "三区三线"、占补平衡 |
| **生态环境管理** | 环评审批、环境监测、应急响应 | 污染因子、环境标准 |
| **农业农村管理** | 振兴规划、质量认证、技术推广 | 乡村振兴、三品一标 |
| **发展规划管理** | 规划编制、经济分析、价格监测 | 宏观经济、产业政策 |
| **城乡建设管理** | 施工许可、质量安全、住房保障 | 建筑标准、房地产监管 |

---

#### 维度3：场景类型

| 场景类型 | 覆盖范围 | 典型场景 |
|---------|---------|---------|
| **办公通用** | 所有领域 | 会议纪要、公文起草、工作报告 |
| **领域业务** | 特定领域 | 用地审批、环评审批、执法检查 |
| **监管巡查** | 监管场景 | 执法检查、安全检查、投诉处理 |

---

### 2.3 分类层级的设计原则

**原则1：认知经济原则**
- 一级分类不超过7±2个（人类短期记忆容量）
- 每个分类名称简洁、直观、易理解

**原则2：功能导向原则**
- 一级分类按功能属性划分（用户关心的"做什么"）
- 二级分类按业务场景划分（用户关心的"怎么做"）

**原则3：领域支撑原则**
- 场景通过标签系统支持领域筛选
- 一个场景可以支持多个领域

**原则4：可扩展原则**
- 新增场景不增加一级分类
- 新增领域不改变分类结构

## 三、场景分类体系设计方案

### 3.1 三级分类体系

```
一级分类（功能分类）：
├── 文档处理
├── 审批服务
├── 规划编制
├── 监管执法
└── 数据分析

二级分类（业务场景）：
每个一级分类下包含若干场景

三级实体（场景智能体）：
每个场景对应一个或多个智能体
```

---

### 3.2 详细分类结构

#### 一级分类：文档处理

```
文档处理
├── 会议纪要（通用智能体）
├── 公文起草（通用智能体）
├── 工作报告（通用智能体）
└── ...其他文档场景
```

**分类依据**：输出成果是文档，核心功能是文档生成

**标签系统**：
- 功能标签：文档生成、格式规范
- 领域标签：支持所有领域（通过领域上下文注入）
- 优先级标签：高频使用

---

#### 一级分类：审批服务

```
审批服务
├── 项目审批（通用智能体）
├── 用地审批（领域智能体）
├── 环评审批（领域智能体）
├── 施工许可（领域智能体）
└── ...其他审批场景
```

**分类依据**：流程导向，核心功能是审批服务

**标签系统**：
- 功能标签：审批、审核、许可
- 领域标签：发改、资规、环保、住建等
- 流程标签：申请、受理、审查、决定

---

#### 一级分类：规划编制

```
规划编制
├── 空间规划（领域智能体）
├── 发展规划（领域智能体）
├── 专项规划（领域智能体）
└── ...其他规划场景
```

**分类依据**：方案导向，核心功能是规划编制

**标签系统**：
- 功能标签：规划、方案、设计
- 领域标签：发改、资规、农业、文旅等
- 类型标签：总体规划、详细规划、专项规划

---

#### 一级分类：监管执法

```
监管执法
├── 执法检查（通用智能体）
├── 安全检查（通用智能体）
├── 投诉处理（通用智能体）
├── 整改验收（通用智能体）
└── ...其他监管场景
```

**分类依据**：监督导向，核心功能是监管执法

**标签系统**：
- 功能标签：检查、执法、处罚、整改
- 领域标签：资规、环保、住建、执法等
- 流程标签：检查、发现、处置、验收

---

#### 一级分类：数据分析

```
数据分析
├── 经济分析（领域智能体）
├── 环境监测（领域智能体）
├── 质量监测（领域智能体）
├── 监测分析（通用智能体）
└── ...其他数据场景
```

**分类依据**：数据导向，核心功能是数据分析

**标签系统**：
- 功能标签：分析、监测、统计、可视化
- 领域标签：发改、环保、教育、农业等
- 类型标签：现状分析、趋势预测、对比分析

## 四、标签系统设计

### 4.1 标签维度

**标签是多维度的，用于场景的筛选和推荐**：

```
标签系统：
├── 功能标签（维度1）
│   ├── 文档生成
│   ├── 审批审核
│   ├── 规划编制
│   ├── 监管执法
│   └── 数据分析
│
├── 领域标签（维度2）
│   ├── 自然资源
│   ├── 生态环境
│   ├── 农业农村
│   ├── 发展改革
│   └── ...其他领域
│
├── 场景类型标签（维度3）
│   ├── 办公通用
│   ├── 领域业务
│   └── 监管巡查
│
├── 优先级标签（维度4）
│   ├── 高频使用
│   ├── 常用
│   └── 低频
│
└── 其他标签（维度5+）
    ├── 专业性强
    ├── 流程复杂
    └── ...其他特征
```

---

### 4.2 标签应用示例

**场景：会议纪要**

### 4.3 标签系统的价值

**价值1：多维度筛选**
```
用户可以通过多个维度筛选场景：
- 选择"文档处理" → 显示3个场景
- 选择"自然资源"领域 → 显示相关场景
- 选择"高频使用" → 显示常用场景
```

**价值2：智能推荐**
```
系统根据用户行为推荐场景：
- 用户经常使用"会议纪要" → 推荐"公文起草"（同属文档处理）
- 用户所在部门是"资规局" → 推荐"用地审批"（自然资源领域）
```

**价值3：统计分析**
```
分析场景使用情况：
- 按功能统计：文档处理使用率40%
- 按领域统计：自然资源场景使用率30%
- 按优先级统计：高频场景使用率80%
```

## 五、优化方案对比

### 5.1 原方案 vs 优化方案

| 维度 | 原方案 | 优化方案 | 改进 |
|------|--------|---------|------|
| **分类层级** | 1层（场景直接作为分类） | 3层（功能→场景→智能体） | ✅ 结构清晰 |
| **一级分类数** | 11个（太多） | 5个（合理） | ✅ 认知负担轻 |
| **分类标准** | 混合（功能+业务） | 统一（功能导向） | ✅ 逻辑清晰 |
| **扩展性** | 差（新增场景增加分类） | 好（新增场景不增加分类） | ✅ 易于扩展 |
| **用户体验** | 选择困难 | 层次清晰 | ✅ 易于使用 |
| **标签系统** | 无 | 完整的多维度标签 | ✅ 支持筛选推荐 |

---

### 5.2 分类数量对比

```
原方案：
一级分类：11个场景（会议纪要、公文起草...）
用户选择：直接从11个中选择 → 困难

优化方案：
一级分类：5个功能分类
├── 文档处理（3个场景）
├── 审批服务（8个场景）
├── 规划编制（5个场景）
├── 监管执法（7个场景）
└── 数据分析（5个场景）

用户选择：
步骤1：选择功能分类（5选1）→ 容易
步骤2：选择具体场景（3-8选1）→ 容易
```

## 七、总结

### 7.1 关键洞察

1. ✅ **分类层级缺失**：原方案缺少功能分类这一层级
2. ✅ **分类逻辑混乱**：场景混合了功能和业务属性
3. ✅ **标签系统缺失**：缺少多维度标签支持筛选和推荐
4. ✅ **用户体验差**：一级分类太多，选择困难

---

### 7.2 优化核心

**核心原则**：
1. **功能导向**：一级分类按功能划分（用户关心的"做什么"）
2. **层级清晰**：功能分类 → 业务场景 → 场景智能体
3. **标签支撑**：多维度标签支持筛选和推荐
4. **易于扩展**：新增场景不增加一级分类
