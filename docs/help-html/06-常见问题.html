❓ 常见问题

安装、配置、使用、安全相关常见问题解答。

🚀 安装部署

启动后无法访问前端页面

按以下步骤排查：

  1. 检查容器：docker ps | grep coapis
  2. 检查端口：docker port coapis-nginx
  3. 检查防火墙：开放 4200 端口 
    firewall-cmd --permanent --add-port=4200/tcp
firewall-cmd --reload
  4. 查看 Nginx 日志：docker logs coapis-nginx
后端服务启动失败

常见原因及解决方法：

  • LLM 服务不可达：检查 COAPIS_LLM_BASE_URL 配置，确保网络连通
  • 端口冲突：lsof -i:8000 检查端口占用
  • 权限问题：chmod 755 /opt/coapis/data
  • 查看日志：docker logs coapis-server
如何升级 CoApis
# 1. 备份
docker compose exec server python backup.py --full

# 2. 拉取最新代码
git pull origin main

# 3. 重新构建
docker compose down
docker compose up -d --build

# 4. 验证
curl http://localhost:8000/api/health

👤 用户管理

如何创建新用户

两种方式：

  • 前端注册：访问 http://<server>:4200/register
  • 管理员创建：设置 → 智能体管理 → 创建用户
用户忘记密码怎么办

管理员重置密码：

curl -X POST http://localhost:8000/api/admin/users/reset-password \
  -H "Authorization: Bearer <admin-token>" \
  -d '{"username": "testuser", "new_password": "newpass123"}'
如何限制用户权限

通过用户等级和角色控制：

等级Token 配额限流
L0100K/月10 req/min
L11M/月10 req/min
L25M/月50 req/min
L320M/月200 req/min
L4无限制1000 req/min

🤖 Agent 配置

如何添加新的 Agent

在 {WORKING_DIR}/system/config.json 中添加：

{
  "agents": {
    "profiles": {
      "new-agent": {
        "model": "gpt-4-turbo",
        "provider": "openai"
      }
    }
  }
}

然后重启：docker compose restart server

Agent 回复很慢怎么办
  • 调整 LLM 并发：COAPIS_LLM_MAX_CONCURRENT=20
  • 启用前缀缓存：vllm serve --enable-prefix-caching
  • 使用更快的模型（参数量更小或量化版本）

📁 文件管理

文件上传大小限制

默认 100MB。可通过 Nginx 调整：

client_max_body_size 500m;
文件上传失败
  1. 检查磁盘空间：df -h /opt/coapis/data
  2. 检查目录权限：ls -la /opt/coapis/data/files
  3. 查看日志：docker logs coapis-server | grep -i upload

🔌 集成配置

如何接入企业微信

控制 → 频道 → 添加频道 → 选择 WeCom，填入 Corp ID、Agent ID、Secret、Token、EncodingAESKey。

如何接入钉钉

控制 → 频道 → 添加频道 → 选择 DingTalk，填入 App Key、App Secret、Verification Code。

如何配置 MCP 客户端
{
  "name": "filesystem",
  "transport": "stdio",
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/data"]
}

🔐 安全相关

如何启用 HTTPS
server {
    listen 443 ssl;
    server_name your-domain.com;
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
}
如何查看审计日志

前端：设置 → 安全 → 审计日志

API：

curl http://localhost:8000/api/audit/logs \
  -H "Authorization: Bearer <token>"
