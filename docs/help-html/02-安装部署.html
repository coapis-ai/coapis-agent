📦 安装部署

三种方式部署 CoApis：一键安装、手动部署、源码安装。

前置要求

配置最低要求推荐配置
CPU4 核8 核+
内存8 GB16 GB+
存储50 GB100 GB+ (SSD)
操作系统Linux (Ubuntu 20.04+ / CentOS 8+ / Debian 11+)
Docker20.10+ 及 Docker Compose 2.0+

方式一：一键安装（推荐）

curl -fsSL https://raw.githubusercontent.com/coapis/coapis/main/install.sh | bash

安装脚本会自动：

  • 检查 Docker 环境
  • 克隆项目代码
  • 生成配置文件
  • 启动所有服务

启动后访问 http://你的服务器IP:4200，用 admin / admin123 登录。

方式二：手动部署

克隆项目
git clone https://github.com/coapis/coapis.git
cd coapis-agent
配置环境变量
cp docker/.env.example docker/.env
vim docker/.env

关键配置项：

# LLM API 服务地址（必填）
COAPIS_LLM_BASE_URL=http://your-llm-server:8082/v1

# 启用认证（建议）
COAPIS_AUTH_ENABLED=True
COAPIS_USER_SYSTEM_ENABLED=True
启动服务
cd docker
docker compose up -d
配置 Agent 和 Provider

编辑 {WORKING_DIR}/system/config.json 和 providers.json。详见 配置指南。

方式三：源码安装

# 克隆
git clone https://github.com/coapis/coapis.git
cd coapis

# 构建前端
cd client && npm ci && npm run build && cd ..

# 配置环境变量
cp docker/.env.example docker/.env

# Docker 构建启动
docker compose -f docker/docker-compose.build.yml up -d --build

LLM 服务配置

使用 Ollama

# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen3.6-27b

# 启动服务
ollama serve

# 配置 CoApis
# docker/.env
COAPIS_LLM_BASE_URL=http://localhost:11434/v1

使用 vLLM

vllm serve \
  --model /path/to/model \
  --tensor-parallel-size 4 \
  --max_num_seqs 8 \
  --max-model-len 131072

# docker/.env
COAPIS_LLM_BASE_URL=http://localhost:8082/v1

使用 OpenAI

# docker/.env
COAPIS_LLM_BASE_URL=https://api.openai.com/v1
OPENAI_API_KEY=sk-...

验证安装

# 检查容器
docker ps | grep coapis

# 健康检查
curl http://localhost:8000/api/health

# 预期输出
# {"status": "healthy", "version": "0.1.0"}

升级 CoApis

# 1. 备份
docker compose exec server python backup.py --full

# 2. 拉取最新代码
git pull origin main

# 3. 重新构建
docker compose down
docker compose up -d --build

# 4. 验证
curl http://localhost:8000/api/health
