💻 命令参考

CoApis 命令行工具（CLI）完整参考手册。

系统初始化

coapis init

初始化工作区和配置文件。

coapis init [OPTIONS]
参数说明
--force覆盖已存在的配置文件
--defaults使用默认值，不进行交互式提示
--accept-security跳过安全确认（配合 --defaults 用于 Docker）
-h, --help显示帮助信息

使用示例：

# 交互式初始化（推荐首次使用）
coapis init

# 非交互式（适用于自动化脚本）
coapis init --defaults --accept-security

# 强制重新初始化
coapis init --force

# 在生产容器中运行
docker exec coapis-server-prod coapis init --defaults --accept-security

管理命令

命令说明
coapis init初始化工作区和配置文件
coapis auth管理 Web 认证
coapis admin管理员系统工具
coapis doctor本地健康检查
coapis models管理 LLM 模型和提供商配置
coapis channels管理频道配置
coapis cron管理定时任务

coapis auth

管理 Web 认证配置。

# 查看认证状态
coapis auth status

# 生成 JWT 密钥
coapis auth generate-key

coapis admin

管理员系统工具。

# 查看系统状态
coapis admin status

# 重置管理员密码
coapis admin reset-password

coapis doctor

本地健康检查，诊断常见问题。

coapis doctor

检查项目：Docker 状态、端口占用、配置文件、LLM 连通性等。

coapis models

管理 LLM 模型和提供商配置。

# 列出所有模型
coapis models list

# 测试模型连通性
coapis models test <model-id>

coapis channels

管理频道配置。

# 列出所有频道
coapis channels list

# 测试频道连接
coapis channels test <channel-id>

coapis cron

管理定时任务。

# 列出所有任务
coapis cron list

# 创建定时任务
coapis cron create --agent-id default --cron "0 9 * * *" --prompt "生成日报"

# 暂停/恢复任务
coapis cron pause <task-id>
coapis cron resume <task-id>

# 手动执行
coapis cron run <task-id>

Docker 管理命令

命令说明
docker compose up -d启动所有服务
docker compose down停止所有服务
docker compose restart server重启后端服务
docker compose logs -f server查看后端日志
docker compose logs -f nginx查看 Nginx 日志
docker ps | grep coapis查看 CoApis 容器状态

Python 初始化器

# 直接调用（在容器内）
python3 -c "
from coapis.system import initialize_system
result = initialize_system()
print(result)
"

# 检查初始化状态
python3 -c "
from coapis.system import check_system_status
import json
print(json.dumps(check_system_status(), indent=2))
"

自动初始化

容器启动时自动检测并执行初始化（通过 entrypoint.sh）：

  1. 检查 config.json, permissions.json, users.json 是否存在
  2. 如果缺失，自动调用 init_workspace.sh
  3. 优先使用 Python 初始化器，回退到 Shell 脚本
