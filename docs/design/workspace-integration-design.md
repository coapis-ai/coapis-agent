# 工作空间与知识库整合设计方案

## 📋 需求背景

**核心需求**：
- 在聊天界面中显示工作空间（包括知识库）
- 方便查看、选择，与聊天界面协同工作
- 提升用户体验，减少页面切换

**用户痛点**：
- 当前工作空间和聊天界面分离，需要频繁切换
- 无法在对话中快速引用工作空间文件
- 知识库管理不够直观

---

## 🎯 设计目标

1. **便捷性**：减少页面切换，提升工作效率
2. **集成性**：聊天与工作空间/知识库深度整合
3. **灵活性**：支持多种使用场景和操作模式
4. **可扩展性**：为未来功能扩展预留空间

---

## 💡 方案对比

### 方案A：聊天界面整合（推荐 ⭐）

#### 设计思路

在聊天界面右侧添加可展开/收起的侧边栏，集成工作空间和知识库功能。

#### 界面布局

```
┌─────────────────────────────────────────────────────────┐
│  Header（智能体选择、设置）                                │
├────────────────┬────────────────────────────────────────┤
│                │                                        │
│  聊天消息区域   │   工作空间侧边栏（可收起）               │
│                │   ├─ 📁 文件列表                        │
│                │   ├─ 🧠 记忆库                          │
│                │   ├─ 📚 知识库                          │
│                │   └─ 🔍 搜索/筛选                        │
│                │                                        │
├────────────────┴────────────────────────────────────────┤
│  输入区域（支持拖拽文件、@引用知识库）                      │
└─────────────────────────────────────────────────────────┘
```

#### 核心功能

**1. 工作空间侧边栏**

```typescript
interface WorkspaceSidebarProps {
  visible: boolean;           // 侧边栏显示状态
  onToggle: () => void;       // 展开/收起
  activeTab: 'files' | 'memory' | 'knowledge';
  onSelectFile: (file: FileItem) => void;
  onSelectKnowledge: (item: KnowledgeItem) => void;
}
```

**功能特性**：
- ✅ 文件树展示（支持展开/折叠）
- ✅ 快速搜索和筛选
- ✅ 拖拽文件到聊天输入框
- ✅ 预览文件内容
- ✅ 记忆库快速查看
- ✅ 知识库条目管理

**2. 输入框增强**

```typescript
interface EnhancedInputProps {
  onDropFile: (files: File[]) => void;
  onMentionKnowledge: (knowledge: KnowledgeItem) => void;
  attachedFiles: File[];
  mentionedKnowledge: KnowledgeItem[];
}
```

**功能特性**：
- ✅ 支持拖拽文件
- ✅ @ 符号触发知识库搜索
- ✅ 显示已附加的文件/知识
- ✅ 快速移除附件

**3. 智能联动**

```typescript
// 示例：在对话中引用文件
用户: "帮我分析这个文件的代码质量"
系统: 自动高亮工作空间侧边栏
用户: 拖拽文件到输入框
系统: 文件附加到消息，智能体自动读取

// 示例：引用知识库
用户: "根据我们的编码规范，检查这段代码"
用户: @编码规范
系统: 显示知识库搜索结果
用户: 选择条目
系统: 知识附加到消息上下文
```

#### 优势

✅ **用户体验好**：
- 无需页面切换，提升效率
- 拖拽操作直观便捷
- 实时预览文件内容

✅ **实现成本低**：
- 复用现有工作空间组件
- 只需添加侧边栏容器
- 改动范围小，风险可控

✅ **灵活性强**：
- 侧边栏可收起，不影响聊天体验
- 支持拖拽、@引用等多种操作
- 易于扩展新功能

✅ **学习成本低**：
- 界面布局常见，用户熟悉
- 操作方式直观（拖拽、@）

#### 劣势

⚠️ **空间受限**：
- 侧边栏宽度有限，不适合复杂操作
- 文件预览可能不够清晰

⚠️ **干扰聊天**：
- 侧边栏展开可能分散注意力
- 需要用户主动管理显示状态

### 方案B：独立工作台

#### 设计思路

设计全新的工作台页面，采用三栏布局，深度整合聊天、工作空间、知识库。

#### 界面布局

```
┌─────────────────────────────────────────────────────────┐
│  工作台 Header（项目切换、快速操作）                        │
├──────────┬──────────────────────────┬───────────────────┤
│          │                          │                   │
│  左侧栏   │      中间对话区           │    右侧工具区      │
│          │                          │                   │
│ 📁 文件   │  ┌──────────────────┐   │  🔧 当前工具       │
│ 🧠 记忆   │  │  消息列表         │   │  ├─ 文件预览       │
│ 📚 知识   │  │                  │   │  ├─ 代码编辑       │
│ 📝 笔记   │  │                  │   │  ├─ 知识编辑       │
│ 🎯 任务   │  └──────────────────┘   │  └─ 输出结果       │
│          │                          │                   │
│          │  ┌──────────────────┐   │                   │
│          │  │  输入框          │   │                   │
│          │  └──────────────────┘   │                   │
│          │                          │                   │
└──────────┴──────────────────────────┴───────────────────┘
```

#### 核心功能

**1. 左侧栏 - 项目导航**

```typescript
interface ProjectSidebarProps {
  projects: Project[];
  activeProject: Project;
  onSelectProject: (project: Project) => void;
  onCreateProject: () => void;
}
```

**功能模块**：
- 📁 **文件管理**：项目文件树，支持上传、编辑、删除
- 🧠 **记忆库**：长期记忆、短期记忆、经验沉淀
- 📚 **知识库**：知识条目、分类管理、搜索查询
- 📝 **笔记**：快速笔记、会议记录
- 🎯 **任务**：任务列表、进度跟踪

**2. 中间区 - 对话中心**

```typescript
interface ConversationCenterProps {
  messages: Message[];
  onSendMessage: (message: string, context: MessageContext) => void;
  context: {
    files: File[];
    knowledge: KnowledgeItem[];
    memories: Memory[];
  };
}
```

**功能特性**：
- ✅ 智能上下文关联（自动识别左侧栏选择的内容）
- ✅ 多模态输入（文本、文件、语音）
- ✅ 流式输出展示
- ✅ 历史对话管理

**3. 右侧栏 - 工作工具**

```typescript
interface WorkToolsProps {
  activeTool: 'preview' | 'editor' | 'output';
  content: ToolContent;
  onUpdate: (content: ToolContent) => void;
}
```

**功能模块**：
- 🔍 **文件预览**：代码高亮、Markdown 渲染、图片预览
- ✏️ **代码编辑**：在线编辑器、语法高亮、自动补全
- 📊 **输出结果**：图表展示、数据分析、导出功能
- ⚙️ **工具配置**：智能体配置、模型选择、参数调整

**4. 智能联动**

```typescript
// 示例：工作流程
用户: 点击左侧"数据分析.py"
系统: 右侧显示代码预览
用户: 输入"帮我优化这个代码"
系统: 智能体自动读取文件内容，分析并给出建议
用户: 查看右侧输出结果
系统: 显示优化后的代码，支持一键应用

// 示例：知识库工作流
用户: 左侧选择"产品需求文档"
系统: 右侧显示文档内容
用户: 输入"根据这份需求，生成测试用例"
系统: 智能体读取文档，生成测试用例
用户: 在右侧查看生成的测试用例，保存到知识库
```

#### 优势

✅ **功能强大**：
- 三栏布局，空间充足
- 支持复杂的工作流程
- 多任务并行处理

✅ **专业性强**：
- 类似 IDE 的界面设计
- 适合深度工作场景
- 支持项目管理

✅ **扩展性好**：
- 易于添加新功能模块
- 支持插件化架构
- 可定制化程度高

#### 劣势

⚠️ **实现成本高**：
- 需要重新设计整个页面
- 开发工作量大
- 测试复杂度高

⚠️ **学习成本高**：
- 界面复杂，新用户需要学习
- 功能太多可能造成困惑
- 需要详细的使用文档和引导

⚠️ **可能影响简洁性**：
- 复杂界面可能让用户感到压力
- 与当前简洁的设计风格冲突
- 需要平衡功能丰富度和易用性

## 🎯 推荐方案

### 推荐：**方案A - 聊天界面整合** ⭐

#### 推荐理由

1. **符合用户习惯**
   - 类似 Slack、Discord 的侧边栏设计
   - 用户无需学习即可使用
   - 操作直观（拖拽、@引用）

2. **实现成本低**
   - 复用现有组件
   - 改动范围小
   - 快速上线验证

3. **渐进式增强**
   - 先实现基础功能，收集反馈
   - 后续可逐步增强
   - 如果需要更强大的功能，再考虑方案B

4. **风险可控**
   - 不影响现有聊天功能
   - 侧边栏可收起，不影响核心体验
   - 易于回滚

#### 实施建议

**阶段一：MVP（1-2周）**
- ✅ 添加工作空间侧边栏（可收起）
- ✅ 文件树展示
- ✅ 拖拽文件到输入框
- ✅ 基础文件预览

**阶段二：增强（2-3周）**
- ✅ @ 符号触发知识库搜索
- ✅ 记忆库快速查看
- ✅ 知识库条目管理
- ✅ 智能上下文关联

**阶段三：优化（持续）**
- ✅ 性能优化
- ✅ 用户反馈迭代
- ✅ 高级功能（版本控制、协作等）

**如果方案A不能满足需求，再考虑方案B**

---

## ✅ 总结

**推荐方案A - 聊天界面整合**，原因：

1. ✅ 实现成本低，快速上线
2. ✅ 用户体验好，学习成本低
3. ✅ 风险可控，易于迭代
4. ✅ 符合用户习惯，无需重新学习

**后续根据用户反馈和实际需求，再评估是否需要方案B的独立工作台。**
