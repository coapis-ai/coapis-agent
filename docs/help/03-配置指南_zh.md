# CoApis 配置指南

## 📋 配置概览

CoApis 采用**三层配置体系**：

```
环境变量 (.env) → 全局配置
Agent 配置 (config.json) → Agent 行为配置
Provider 配置 (providers.json) → LLM 服务配置
```

## 🔧 环境变量配置

配置文件位置：`docker/.env`

> **重要**: `.env` 是单一配置源，`docker-compose.yaml` 中的变量引用来自此文件。

### 核心配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `COAPIS_LLM_BASE_URL` | string | - | LLM API 服务地址（**必填**） |
| `COAPIS_AUTH_ENABLED` | bool | `True` | 是否启用认证系统 |
| `COAPIS_USER_SYSTEM_ENABLED` | bool | `True` | 是否启用多租户用户体系 |
| `COAPIS_WORKING_DIR` | string | `/apps/ai/coapis` | 工作目录（所有数据根路径） |
| `COAPIS_PORT` | int | `8000` | 后端容器内端口 |
| `COAPIS_FRONTEND_PORT` | int | `4200` | 前端 Nginx 宿主机端口 |

### 目录结构说明

```
{WORKING_DIR}/
├── system/              ← 系统级文件（全局共享）
│   ├── config.json      ← Agent 配置
│   ├── providers.json   ← Provider 配置
│   ├── users.json       ← 用户数据
│   ├── auth.json        ← 认证配置
│   ├── permissions.json ← 权限配置
│   ├── evolution_config.json
│   ├── token_usage.json
│   └── .secret/         ← 密钥存储
├── workspaces/          ← 用户级数据（按用户隔离）
│   └── {username}/      ← 每个用户独立工作区
├── agents/              ← 全局共享 Agent 模板（只读）
├── skills/              ← 全局共享技能模板（只读）
└── logs/                ← 运行日志
```

### LLM 限流配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `COAPIS_LLM_MAX_CONCURRENT` | int | `10` | 最大并发 LLM 调用数 |
| `COAPIS_LLM_MAX_QPM` | int | `600` | 每分钟最大查询数（0=无限制） |
| `COAPIS_LLM_MAX_RETRIES` | int | `3` | 每次调用最大重试次数 |
| `COAPIS_LLM_BACKOFF_BASE` | float | `1.0` | 退避基础时间（秒） |
| `COAPIS_LLM_BACKOFF_CAP` | float | `10.0` | 退避最大时间（秒） |
| `COAPIS_LLM_RATE_LIMIT_PAUSE` | float | `5.0` | 429 错误暂停时间（秒） |
| `COAPIS_LLM_ACQUIRE_TIMEOUT` | float | `300.0` | 获取并发槽超时时间（秒） |

## 🤖 Agent 配置

配置文件位置：`{WORKING_DIR}/system/config.json`

> **注意**: 首次启动时，`init_workspace.sh` 会自动创建基础 `config.json`。你可以根据需要修改。

### 基本结构

```json
{
  "agents": {
    "active_agent": "default",
    "profiles": {
      "default": {
        "model": "qwen3.6-27b",
        "provider": "local_llm",
        "enable_memory": false,
        "tool_calling_method": "raw"
      }
    }
  }
}
```

### 配置字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `active_agent` | string | ✅ | 默认激活的 Agent ID |
| `profiles` | object | ✅ | Agent 配置集合 |
| `model` | string | ✅ | 模型名称（必须与 LLM 服务实际提供的一致） |
| `provider` | string | ✅ | Provider ID（必须与 providers.json 中的 ID 匹配） |
| `enable_memory` | bool | - | 是否启用记忆模块（Qwen3 建议 `false`） |
| `tool_calling_method` | string | - | 工具调用方式（Qwen3 建议 `raw`） |

### 多 Agent 配置示例

```json
{
  "agents": {
    "active_agent": "default",
    "profiles": {
      "default": {
        "model": "qwen3.6-27b",
        "provider": "local_llm",
        "enable_memory": false,
        "tool_calling_method": "raw"
      },
      "coder": {
        "model": "deepseek-coder-v2",
        "provider": "openai",
        "enable_memory": true,
        "tool_calling_method": "auto"
      },
      "analyst": {
        "model": "gpt-4-turbo",
        "provider": "openai",
        "enable_memory": true,
        "tool_calling_method": "auto"
      }
    }
  }
}
```

## 🔌 Provider 配置

配置文件位置：`{WORKING_DIR}/system/providers.json`

### 基本结构

```json
{
  "provider_id": {
    "id": "provider_id",
    "type": "openai|anthropic|google",
    "api_base": "http://llm-service:port/v1",
    "api_key": "your-api-key"
  }
}
```

### 配置字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | Provider 唯一标识（如 `local_llm`） |
| `type` | string | ✅ | Provider 类型：`openai`、`anthropic`、`google` |
| `api_base` | string | ✅ | API 基础地址（从容器内可访问） |
| `api_key` | string | ✅ | API 密钥（本地 LLM 推理服务无需认证时用 `none`） |

### 常见 Provider 配置示例

**本地 LLM 推理服务（无需认证）：**

## 🎯 性能调优配置

### ContextCompressor 阈值

在 `server/coapis/agent/context_compressor.py` 中调整：

```python
HISTORY_TOKEN_BUDGET = 500  # 历史消息 Token 预算（超过则触发压缩）
TIER1_THRESHOLD = 5         # Tier1 压缩触发阈值（轮数）
```

### 本地 LLM 推理服务启动参数建议（以 vLLM 为例）

```bash
vllm serve \
  --model /path/to/model \
  --tensor-parallel-size 4 \
  --max_num_seqs 8 \
  --max-model-len 131072 \
  --max-num-batched-tokens 8192 \
  --block-size 16 \
  --gpu-memory-utilization 0.87 \
  --enable-prefix-caching \
  --enable_chunked_prefill \
  --enforce_eager
```

## 📝 配置优先级

配置的优先级从高到低：

1. **环境变量**（`.env` 文件或系统环境变量）
2. **配置文件**（`config.json`、`providers.json`）
3. **内置默认值**

## 🔍 配置验证

启动后检查配置是否正确：

```bash
# 检查后端健康状态
curl http://localhost:8000/api/health

# 查看后端日志确认配置加载
docker compose logs server | grep -i "config\|provider\|agent"
```
