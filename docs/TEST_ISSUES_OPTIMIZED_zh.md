#### A-06 | 🔴 Bug | P0

**问题描述**: 智能体无法读取"我的空间"文件，一直卡在扫描阶段

**代码位置**: `app/runner/runner.py`（`_persist_chat_messages`），`app/routers/files.py`，`app/crons/`/`app/tools/`（ToolRegistry）

**根因分析**:  
这是 workspace 文件服务与 LLM 工具的**集成断裂**问题。代码验证发现：
- `files.py` 有完整的文件管理 API（upload/download/preview/rename/delete/mkdir）
- 但 `runner.py` 的消息持久化链路中，**没有找到将文件内容注入 LLM 上下文的逻辑**
- "卡在扫描阶段"表明前端发起了文件读取请求，但后端没有对应的工具/接口来响应

核心问题：**ToolRegistry 中可能未注册文件读取工具**，或者 LLM agent 的工具列表中不包含文件操作能力。

**修复建议**:  
1. 检查 ToolRegistry 是否注册了文件读取相关工具（`read_file` / `get_file_content`）
2. 如果已注册，检查 LLM agent 的 tool_config 是否包含该工具
3. 在 `runner.py` 中添加文件读取到 LLM 上下文的桥接逻辑
4. **短期方案**: 在 chat message 中传递文件路径，runner 解析后读取文件内容并注入 context

**修复状态**: ❌ 未修复  
**负责人建议**: 后端开发（核心功能，需架构评审）

#### D-01 | ⚪ Not Reproducible | P0

**问题描述**: 定时任务到时间后没有执行

**代码位置**: `app/crons/manager.py`（`run_job` / `run_job_once` / `_scheduled_callback`），`app/crons/executor.py`

**根因分析**:  
代码验证发现完整的定时任务框架：
- `manager.py` 有 `run_job` / `run_job_once` 方法

可能原因：

**修复建议**:
