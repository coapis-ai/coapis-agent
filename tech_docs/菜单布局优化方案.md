# CoApis 菜单布局优化方案

> 版本：v1.1  
> 日期：2026-07-21  
> 范围：菜单结构重组 + 权限配置调整  
> 不涉及：模块功能改造

---

## 一、优化目标

### 1.1 核心目标

1. **降低门槛** - 普通用户只看到必要菜单，减少认知负担
2. **优化体验** - 管理员快速找到配置入口
3. **权限清晰** - 不同角色看到不同菜单，支持个人资源管理

### 1.2 当前问题

```
当前菜单（22项，平铺）：
├─ 首页
├─ 场景
├─ 对话
├─ 智能体 → 模板、技能、工具、MCP
├─ 模型
├─ 渠道
├─ 定时任务
├─ 用户
├─ 审计
├─ 分析
├─ 设置 → 安全、其他设置
└─ ...

问题：
- 普通用户看到22个菜单，不知道用哪个
- 技术术语（MCP、进化）让普通用户困惑
- 管理员配置入口分散
```

## 二、资源所有权模型

### 2.1 核心概念

```
资源分类：
├─ 个人资源（用户可完全控制）
│   ├─ 个人智能体
│   ├─ 个人定时任务
│   ├─ 个人渠道配置
│   ├─ 个人文件
│   └─ 个人会话
│
└─ 系统资源（管理员控制）
    ├─ 系统智能体模板
    ├─ 系统技能包
    ├─ 系统工具
    ├─ MCP服务
    ├─ 模型配置
    ├─ 用户管理
    └─ 系统设置
```

### 2.2 权限范围

| 权限范围 | 说明 | 示例 |
|---------|------|------|
| `all` | 所有资源（管理员） | 所有智能体、所有任务 |
| `self` | 个人资源（普通用户） | 自己创建的智能体、自己的任务 |
| `none` | 无权限 | 不可见或只读 |

## 三、菜单重组方案

### 3.1 双入口模式

```
用户访问系统
    ↓
默认进入工作台（普通用户）/ 控制台（管理员）
    ↓
可通过按钮切换入口
```

### 3.2 工作台菜单（面向普通用户）

**菜单项：5个**

```
工作台（/workbench）
├─ 首页          ← 默认页，展示常用场景、最近使用
├─ 场景          ← 浏览和使用场景
├─ 对话          ← 查看历史会话
├─ 文件          ← 个人文件管理
└─ 设置          ← 个人偏好设置（模型选择、个人智能体、个人任务、个人渠道）
```

**特点**：
- ✅ 菜单项少，一目了然
- ✅ 命名通俗，无需技术背景
- ✅ 一级菜单，无需展开
- ✅ 支持个人资源管理（智能体、定时任务）
- ✅ 5秒内找到目标功能

### 3.3 控制台菜单（面向管理员）

**菜单项：12个（含子菜单）**

```
控制台（/console）
├─ 概览          ← Dashboard，系统统计
├─ 场景          ← 场景管理（CRUD）
├─ 智能体        ← 子菜单展开
│   ├─ 模板      ← 系统智能体模板管理
│   ├─ 技能      ← 技能包管理
│   ├─ 工具      ← 工具管理
│   └─ MCP       ← MCP服务配置
├─ 模型          ← 模型配置管理
├─ 渠道          ← 渠道管理（所有用户的渠道）
├─ 定时任务      ← 定时任务管理（系统级 + 用户级）
├─ 用户          ← 用户管理
├─ 分析          ← 使用数据分析
├─ 审计          ← 操作日志查看
└─ 设置          ← 系统配置
    ├─ 安全      ← 安全设置
    └─ 其他      ← 其他配置
```

**特点**：
- ✅ 功能完整，专业可控
- ✅ 层级分明，逻辑清晰
- ✅ 概览入口，快速了解系统状态
- ✅ 最多2级，操作简单

### 6.2 权限范围过滤

```typescript
// client/src/hooks/useResourceScope.ts

import { useQuery } from '@tanstack/react-query';
import { api } from '@/api';

/**
 * 获取用户可见的资源列表（根据权限范围）
 * @param resource 资源类型（agents, cron_tasks）
 * @param scope 权限范围（all, self）
 */
export function useResourceScope(resource: string, scope: 'all' | 'self' = 'self') {
  const { data, isLoading } = useQuery({
    queryKey: [resource, scope],
    queryFn: async () => {
      const response = await api.get(`/${resource}`, {
        params: { scope }
      });
      return response;
    },
  });
  
  return { data, isLoading };
}

// 使用示例
function MyCronTasks() {
  const { data: tasks } = useResourceScope('cron_tasks', 'self');
  
  return (
    <div>
      {tasks?.map(task => (
        <TaskCard key={task.id} task={task} />
      ))}
    </div>
  );
}
```

## 七、后端实现

### 7.1 资源范围过滤API

```python
# server/coapis/app/routers/cron.py

from fastapi import APIRouter, Depends, Query
from ..auth import get_current_user
from ..models.user import User

router = APIRouter()

@router.get("/cron")
async def list_cron_tasks(
    scope: str = Query("all", description="权限范围: all, self"),
    user: User = Depends(get_current_user)
):
    """获取定时任务列表"""
    from ..services.cron_service import CronService
    
    cron_service = CronService()
    
    # 管理员可以看到所有任务
    if user.role in ["admin", "super_admin"]:
        if scope == "all":
            return cron_service.get_all_tasks()
        else:
            return cron_service.get_user_tasks(user.id)
    
    # 普通用户只能看到自己的任务
    return cron_service.get_user_tasks(user.id)
```

### 7.2 资源所有权检查

```python
# server/coapis/services/cron_service.py

class CronService:
    def get_user_tasks(self, user_id: str):
        """获取用户的定时任务"""
        return self.db.query(CronTask).filter(
            CronTask.created_by == user_id
        ).all()
    
    def update_task(self, task_id: str, task_data: dict, user_id: str, user_role: str):
        """更新定时任务"""
        task = self.db.query(CronTask).filter(CronTask.id == task_id).first()
        
        if not task:
            raise HTTPException(404, "Task not found")
        
        # 权限检查：管理员可以修改所有任务，普通用户只能修改自己的任务
        if user_role not in ["admin", "super_admin"]:
            if task.created_by != user_id:
                raise HTTPException(403, "No permission to modify this task")
        
        # 更新任务
        for key, value in task_data.items():
            setattr(task, key, value)
        
        self.db.commit()
        return task
```
