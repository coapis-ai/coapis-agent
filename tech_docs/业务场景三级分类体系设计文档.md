# 业务场景三级分类体系设计文档

> 创建时间：2026-07-22  
> 版本：v1.0

---

## 📋 概述

本文档定义了CoApis智能体平台的业务场景三级分类体系，支持场景的多标签交叉，实现灵活的场景管理和检索。

---

## 🏗️ 分类架构

### 三级分类结构

```
一级分类（dimension）
  ├── 二级分类（category）
  │     ├── 三级分类（具体场景）
  │     │     └── 场景实例（scene）
  │     └── 场景实例（scene）
  └── 二级分类（category）
        └── 场景实例（scene）
```

### 标签类型

| 标签类型 | 英文 | 说明 | 示例 |
|---------|------|------|------|
| **维度标签** | dimension | 一级分类，按服务对象划分 | 政务服务、企业服务、教育服务 |
| **分类标签** | category | 二级分类，按业务类型划分 | 行政审批、监管执法、公共服务 |
| **行业标签** | industry | 行业属性标签，支持交叉 | 土地资源、环境保护、农业 |
| **频率标签** | frequency | 使用频率标签 | 高频、中频、低频 |

## 🏛️ 一级分类（dimension）

### 1. 办公通用（nature）

**适用范围**：所有行业、所有岗位的通用办公场景

**特点**：
- 通用性强，无需专业知识背景
- 高频使用，日常必用
- 即开即用，快速上手

**包含场景**：48个

---

### 2. 政务服务（government-service）

**适用范围**：政府机关业务场景

**特点**：
- 政务专用，符合行政审批流程
- 行业明确，覆盖国土、环保、农业等
- 交叉标签丰富，支持多维度检索

**包含场景**：27个

---

### 3. 企业服务（enterprise-service）

**适用范围**：企业经营和管理场景

**特点**：
- 企业专用，支持经营决策
- 业务导向，覆盖管理、营销等

**包含场景**：（待扩展）

---

### 4. 教育服务（education-service）

**适用范围**：学校和教育机构业务场景

**包含场景**：4个

**二级分类**：
- 教学管理
- 招生管理
- 教师发展

---

### 5. 医疗健康（healthcare-service）

**适用范围**：医疗机构和公共卫生场景

**包含场景**：2个

**二级分类**：
- 医疗服务
- 疾病防控

---

### 6. 按领域分类（domain）

**说明**：保留的原有分类体系，用于兼容性

## 🔀 交叉标签示例

### 示例1：环评审批

```json
{
  "id": "eia-approval",
  "name": "环评审批",
  "primary_tag_id": "admin-approval",
  "tag_ids": [
    "admin-approval",      // 二级分类：行政审批
    "environment-industry" // 行业标签：环境保护

}
```

- 主分类：行政审批（政务服务）
- 行业属性：环境保护

## 💡 设计原则

### 1. 通用性优先

- 办公场景适用所有行业
- 业务场景按行业细分

### 2. 多维度检索

- 一级分类：按服务对象
- 二级分类：按业务类型
- 行业标签：按行业属性
- 频率标签：按使用频率

### 3. 灵活扩展

- 标签可新增
- 场景可交叉
- 分类可调整

### 4. 符合认知

- 对齐商业产品（钉钉、飞书、企业微信）
- 用户友好命名
- 清晰的层级关系
