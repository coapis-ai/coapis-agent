# 模型管理 UI 设计补充方案

> **目标**: 为模型类型扩展功能设计完整的前端管理界面
> 
> **创建时间**: 2026-07-18
> 
**优先级**: P0（用户体验关键）

---

## 一、当前问题

### 1.1 缺失的前端功能

- ❌ 无法在界面上区分不同类型的模型
- ❌ 无法筛选/查看特定类型的模型
- ❌ 无法通过 UI 配置模型类型和参数
- ❌ 依赖手动编辑 JSON 文件

### 1.2 用户痛点

**场景 1：添加嵌入模型**
```
用户想要：点击"添加模型" → 选择"嵌入模型" → 输入配置 → 测试 → 保存
当前体验：手动编辑 JSON 文件 → 重启服务 → 猜测是否生效
```

**场景 2：选择嵌入模型**
```
用户想要：在知识库设置中选择"使用哪个嵌入模型"
当前体验：无法选择，系统不知道有哪些嵌入模型可用
```

## 四、实现建议

### 4.1 后端 API 扩展

```python
# 新增 API 端点

# 1. 获取指定类型的模型
GET /api/models?type=embedding

# 2. 测试模型连接
POST /api/models/{model_id}/test
{
  "test_input": "测试文本"
}

# 3. 设置默认模型
POST /api/models/default
{
  "chat": "gpt-4o",
  "embedding": "text-embedding-3-large"
}

# 4. 获取模型使用统计
GET /api/models/{model_id}/stats
```

### 4.2 前端组件结构

```
client/src/pages/Settings/
├── ModelManagement/
│   ├── index.tsx              # 主页面
│   ├── ModelList.tsx          # 模型列表
│   ├── ModelFilter.tsx        # 筛选器
│   ├── ModelDetail.tsx        # 模型详情
│   ├── AddModelWizard.tsx     # 添加模型向导
│   ├── ModelTypeTag.tsx       # 类型标签组件
│   └── DefaultModelSettings.tsx # 默认模型设置
```
