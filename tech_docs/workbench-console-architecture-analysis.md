# 工作台与控制台架构深度分析

> **核心问题**：工作台和控制台的关系是什么？用户如何理解和使用？  
> **创建日期**：2026-07-17

### 1.3 核心问题

**问题1：菜单不一致**
- 工作台菜单：场景分类（办公、数据、文档）
- 控制台菜单：系统功能（聊天、控制、智能体、设置）
- 用户点击场景卡片 → 跳转聊天 → 菜单变化

**问题2：导航混淆**
- 用户在工作台选择"办公"场景
- 点击"会议纪要"卡片
- 跳转到聊天页面
- 左侧菜单变成控制台菜单
- 用户如何返回工作台？

**问题3：心智模型**
- 工作台 = 用户日常工作的入口？
- 控制台 = 系统管理？
- 两者是什么关系？

## 二、方案分析

### 方案A：完全独立的两个系统

**架构**：
```
┌──────────────────────┐       ┌──────────────────────┐
│    工作台             │       │    控制台            │
│  (basename: /workbench)│     │  (basename: /console)│
├──────────┬───────────┤       ├──────────┬───────────┤
│ 场景菜单 │ 场景卡片  │       │ 系统菜单 │ 各个页面  │
└──────────┴───────────┘       └──────────┴───────────┘
```

**交互**：
```
工作台 → 点击场景卡片 → 新窗口打开聊天
                       或
工作台 → 点击场景卡片 → 控制台聊天页面
```

**优点**：
- ✅ 工作台和控制台职责清晰
- ✅ 工作台专注业务场景
- ✅ 控制台专注系统管理

**缺点**：
- ❌ 两个系统，维护成本高
- ❌ 新窗口打开，用户体验不连贯
- ❌ 控制台聊天，菜单变化

---

- ❌ 工作台页面没有自己的菜单（场景分类）

## 四、推荐方案：方案D

### 4.1 核心设计

**产品定位**：
- **控制台** = 统一的工作平台
- **聊天** = 核心功能
- **工作台** = 聊天的场景入口

**用户心智**：
- 用户打开系统 → 默认进入聊天页面
- 用户想使用特定场景 → 点击"工作台" Tab → 选择场景 → 开始对话
- 用户想查看历史 → 点击"历史会话" Tab
- 用户想开始新对话 → 点击"新会话" Tab

**交互流程**：
```
用户登录系统
    ↓
默认进入聊天页面（显示工作台 Tab）
    ↓
┌─ 如果用户选择场景 ─┐
│  点击"工作台" Tab  │
│  ↓                 │
│  选择场景卡片      │
│  ↓                 │
│  切换到"新会话" Tab│

┌─ 如果用户查看历史 ─┐
```

### 4.3 技术实现

#### 4.3.1 路由设计

**现有路由**：
```
/console/chat           → 聊天页面
/console/channels       → 通道页面
/console/sessions       → 会话页面
...
```

**新增路由**：
```
/console/chat           → 聊天页面（默认显示工作台Tab）
/console/chat?tab=history → 聊天页面（显示历史会话Tab）
/console/chat?tab=new   → 聊天页面（显示新会话Tab）
/console/chat/:chatId   → 聊天页面（显示具体会话，tab=history）
/console/chat?scene=:sceneId → 聊天页面（代入场景，tab=new）
```

#### 4.3.2 组件设计

**文件位置**：`client/src/pages/Chat/index.tsx`

```typescript
export default function ChatPage() {
  const [searchParams, setSearchParams] = useSearchParams();
  const [activeTab, setActiveTab] = useState('workbench');
  const [scene, setScene] = useState(null);
  const [chatId, setChatId] = useState(null);
  
  // 从 URL 参数读取状态
  useEffect(() => {
    const tab = searchParams.get('tab') || 'workbench';
    const sceneId = searchParams.get('scene');
    const id = searchParams.get('chatId');
    
    setActiveTab(tab);
    
    if (sceneId) {
      loadScene(sceneId);
    }
    
    if (id) {
      setChatId(id);
    }
  }, [searchParams]);
  
  // Tab切换
  const handleTabChange = (tab: string) => {
    setActiveTab(tab);
    setSearchParams({ tab });
  };
  
  // 场景代入
  const handleSceneSelect = async (sceneId: string) => {
    const result = await scenesApi.injectScene(sceneId);
    setChatId(result.chat_id);
    setScene(result.scene);
    setActiveTab('new');
    setSearchParams({ tab: 'new', chatId: result.chat_id });
  };
  
  return (
    <div className="chat-page">
      {/* 顶部Tab */}
      <ChatTabs 
        activeKey={activeTab}
        onChange={handleTabChange}
      />
      
      {/* 内容区 */}
      {activeTab === 'workbench' && (
        <Workbench onSceneSelect={handleSceneSelect} />
      )}
      
      {activeTab === 'history' && (
        <ChatHistory onSelectChat={handleSelectChat} />
      )}
      
      {activeTab === 'new' && (
        <ChatContent 
          chatId={chatId}
          scene={scene}
          onExitScene={handleExitScene}
        />
      )}
    </div>
  );
}
```

```typescript
interface Props {
```

```typescript
export default function Workbench({ onSceneSelect }: Props) {
  const [scenes, setScenes] = useState({});
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    loadAllScenes();
  }, []);
  
  const loadAllScenes = async () => {
    setLoading(true);
    const tags = await scenesApi.getWorkbenchMenu();
    const scenesData = {};
    
    for (const tag of tags) {
      const res = await scenesApi.getWorkbenchSection(tag.id);
      scenesData[tag.id] = {
        tag: res.tag,
        scenes: res.scenes
      };
    }
    
    setScenes(scenesData);
    setLoading(false);
  };
  
  if (loading) {
    return <Spin />;
  }
  
  return (
    <div className="workbench">
      {Object.values(scenes).map(({ tag, scenes }: any) => (
        <div key={tag.id} className="scene-section">
          <h3 className="section-title">
            {tag.icon} {tag.name}
          </h3>
          
          <div className="scene-grid">
            {scenes.map(scene => (
              <SceneCard
                key={scene.id}
                scene={scene}
                onClick={() => onSceneSelect(scene.id)}
              />
            ))}
          </div>
        </div>
      ))}
    </div>
  );
}
```

### 5.1 推荐方案

**方案D：工作台作为聊天页面的入口（Tab切换）**

### 5.2 核心设计

|------|------|
| **产品定位** | 工作台 = 聊天的场景入口 |
| **UI布局** | 聊天页面顶部Tab：工作台、历史会话、新会话 |
