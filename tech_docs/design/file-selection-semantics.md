# 文件选择的产品语义设计

## 🎯 核心理念

### 选中文件夹
- **工作路径**：设置对话的工作目录
- **上下文范围**：AI 可以访问这个文件夹下的文件
- **不会自动加载**：AI 不会自动读取所有文件内容
- **按需读取**：用户明确要求时才读取具体文件

### 选中文件
- **明确引用**：文件是对话的一部分
- **自动参考**：AI 会自动读取文件内容作为背景信息
- **智能使用**：AI 根据用户问题判断如何使用文件
  - 参考模式：作为背景知识、参考资料
  - 使用模式：处理、修改、分析文件内容

---

## 💡 使用场景

### 场景1：参考文件

**选中**：📄 report.pdf

**用户问题**："总结这个报告的主要内容"

**AI 行为**：
1. 自动读取 report.pdf 内容
2. 总结报告要点
3. 不修改原文件

---

### 场景2：使用文件

**选中**：📄 data.csv

**用户问题**："帮我分析这个数据并生成图表"

**AI 行为**：
1. 读取 data.csv 内容
2. 分析数据结构
3. 生成可视化图表
4. 可能输出新文件

---

### 场景3：多文件处理

**选中**：
- 📄 file1.md
- 📄 file2.md
- 📄 file3.md

**用户问题**："合并这三个文件"

**AI 行为**：
1. 读取所有选中文件
2. 合并内容
3. 生成新文件

---

### 场景4：文件夹 + 文件混合

**选中**：
- 📁 docs/           ← 工作路径
- 📄 README.md       ← 具体文件

**用户问题**："根据 docs 目录下的文档，完善 README.md"

**AI 行为**：
1. 设置工作路径为 docs/
2. 读取 README.md 内容
3. 扫描 docs/ 目录结构
4. 根据需要读取相关文档
5. 更新 README.md

---

## 🔧 技术实现

### 前端传递给后端的数据

```json
{
  "selected_files": [
    {
      "id": "/media/report.pdf",
      "name": "report.pdf",
      "type": "file",
      "usage": "auto"  // 自动判断：参考或使用
    }
  ],
  "selected_folders": [
    {
      "id": "/media/docs",
      "name": "docs",
      "type": "folder",
      "usage": "context"  // 工作路径/上下文
    }
  ]
}
```

### AI 提示词模板

```
用户选择了以下文件：
- report.pdf (文件)
- data.csv (文件)

用户选择了以下工作路径：
- docs/ (文件夹)

说明：
- 选中的文件可能需要参考或使用，请根据用户问题灵活处理
- 工作路径下的文件可以作为背景知识，需要时可以访问
- 尽量使用选中的文件来回答用户的问题

用户问题：{user_question}
```

## 🎨 UI 优化建议

### 方案1：简单模式（推荐）

**已选文件区域**：
```
已选择 3 项

📁 1 个工作路径
   docs/ ← 设置对话工作目录

📄 2 个文件
   report.pdf ← 自动参考/使用
   data.csv   ← 自动参考/使用
```

**AI 行为**：
- 自动读取选中文件内容
- 根据问题判断是参考还是使用
- 无需用户额外指定用途

---

### 方案2：高级模式（可选）

**已选文件区域**：
```
已选择 3 项

📁 1 个工作路径
   docs/

📄 2 个文件
   ☑️ report.pdf [参考]  ← 下拉选择
   ☑️ data.csv   [使用]  ← 下拉选择
```

**用途选项**：
- **参考**：作为背景信息，不修改文件
- **使用**：需要处理、修改文件内容
- **自动**：AI 根据问题判断（默认）

---

## ✅ 推荐方案

**采用简单模式**：
1. 选中文件夹 = 工作路径
2. 选中文件 = 自动参考/使用
3. AI 根据用户问题智能判断
4. 无需用户额外指定用途

**优点**：
- 交互简单，用户无需额外操作
- AI 足够智能，可以判断用途
- 符合用户直觉
