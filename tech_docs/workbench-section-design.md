# 工作台一级分区设计

> **核心思路**：直接按标签分区展示场景  
> **极简设计**：无筛选器、无Tab切换  
> **创建日期**：2026-07-17

---

## 一、核心设计

### 1.1 一级分区概念

**工作台直接展示标签分区**：

```
工作台
├── 快捷功能区（横向滚动）
│   └── 用户常用场景
│
├── 📁 办公
│   ├── 会议纪要
│   ├── 工作报告
│   ├── 邮件撰写
│   ├── 通知公告
│   └── 工作计划
│
├── 📊 数据分析
│   ├── 数据统计
│   ├── 图表生成
│   └── 报表导出
│
├── 📄 文档处理
│   ├── 政策解读
│   ├── 合同审查
│   └── 公文起草
│
└── ...更多分区
```

**核心要点**：
- ✅ 直接按标签分区展示
- ✅ 不需要筛选器
- ✅ 不需要Tab切换
- ✅ 用户一眼看到所有分类和场景

---

### 1.2 为什么这样设计？

**简单直接**：
- 用户进入工作台，直接看到所有场景
- 不需要操作筛选器
- 不需要理解复杂的筛选逻辑

**符合用户心智**：
- 用户看到"办公"区，就知道里面有办公相关的场景
- 用户看到"数据分析"区，就知道里面有数据分析的场景
- 自然、直观

## 三、展示逻辑

### 3.1 哪些标签作为分区展示？

**原则**：
- 只展示"功能标签"维度的标签
- 不展示"行业标签"维度的标签
- 不展示"场景类型"、"使用频率"等维度

**原因**：
- 功能标签是用户最常用的分类方式
- 行业标签可以作为场景的附属属性
- 避免分区太多，用户需要滚动太多

**示例**：

```
展示的分区（功能标签）：
├── 📁 办公
├── 📊 数据分析
├── 📄 文档处理
└── 💬 沟通协作

不展示为分区的标签：
├── 🌐 通用（行业标签）
├── 🏛️ 政府（行业标签）
├── ⭐ 高频（频率标签）
└── 🔒 专业（类型标签）
```

---

### 3.2 场景归属哪个分区？

**规则**：
- 场景的"功能标签"决定它归属哪个分区
- 一个场景只能归属一个功能分区
- 如果场景有多个功能标签，选择第一个（或最重要的）

**示例**：

```
会议纪要场景：
├── 功能标签：办公（主要）、文档处理（次要）
├── 行业标签：通用
└── 归属分区：📁 办公

政策解读场景：
├── 功能标签：文档处理
├── 行业标签：政府
└── 归属分区：📄 文档处理
```

---

### 3.3 分区排序

**排序规则**：
- 按标签的 `sort_order` 降序排序
- 或者按标签关联的场景数量排序
- 或者按标签使用频率排序

**示例**：

```
按 sort_order 排序：
1. 📁 办公（sort_order: 100）
2. 📊 数据分析（sort_order: 90）
3. 📄 文档处理（sort_order: 80）
4. 💬 沟通协作（sort_order: 70）
```

---

### 3.4 场景卡片排序

**排序规则**：
- 按场景使用频率排序（高频在前）
- 或者按场景创建时间排序（新场景在前）
- 或者按场景名称排序

**示例**：

```
📁 办公分区：
├── 会议纪要（使用 100 次）
├── 工作报告（使用 80 次）
├── 邮件撰写（使用 60 次）
└── 通知公告（使用 40 次）
```

## 四、数据结构

### 4.1 标签配置

**tags.json**：

```json
{
  "tags": [
    {
      "id": "office-function",
      "name": "办公",
      "icon": "📁",
      "dimension": "function",
      "sort_order": 100,
      "enabled": true,
      "show_as_section": true,  // 是否作为分区展示
      "keywords": ["会议", "报告", "邮件"],
      "related_skills": ["audio-transcription", "content-extraction"]
    },
    {
      "id": "data-analysis-function",
      "name": "数据分析",
      "icon": "📊",
      "dimension": "function",
      "sort_order": 90,
      "enabled": true,
      "show_as_section": true,
      "keywords": ["数据", "统计", "图表"],
      "related_skills": ["data-processing", "chart-generation"]
    },
    {
      "id": "general-industry",
      "name": "通用",
      "icon": "🌐",
      "dimension": "industry",
      "sort_order": 100,
      "enabled": true,
      "show_as_section": false,  // 不作为分区展示
      "keywords": []
    }
  ]
}
```

**关键字段**：
- `dimension`: 标签维度（function、industry、type、frequency）
- `show_as_section`: 是否作为分区展示（只有 function 维度的标签为 true）

---

### 4.2 场景配置

**scenes.json**：

```json
{
  "scenes": [
    {
      "id": "meeting-minutes",
      "name": "会议纪要",
      "icon": "📝",
      "description": "快速生成会议纪要，支持音频转写",
      "short_description": "支持音频转写",  // 简短描述，显示在卡片上
      
      "tags": [
        "office-function",      // 功能标签（决定分区）
        "general-industry",     // 行业标签
        "high-frequency"        // 频率标签
      ],
      
      "primary_tag": "office-function",  // 主标签（决定归属分区）
      
      "system_prompt": "你是一个专业的会议纪要助手...",
      "welcome_message": "我可以帮您快速生成会议纪要...",
      
      "enabled": true
    }
  ]
}
```

**关键字段**：
- `tags`: 场景的所有标签
- `primary_tag`: 主标签（决定场景归属哪个分区）

### 5.2 服务层实现

```python
def get_workbench_data() -> dict:
    """获取工作台数据"""
    
    # 1. 获取快捷访问场景
    quick_access = get_quick_access_scenes()
    
    # 2. 获取所有功能标签（作为分区）
    function_tags = get_tags_by_dimension("function")
    function_tags = [t for t in function_tags if t["show_as_section"]]
    function_tags.sort(key=lambda t: t["sort_order"], reverse=True)
    
    # 3. 构建分区数据
    sections = []
    for tag in function_tags:
        # 获取该标签下的所有场景
        scenes = get_scenes_by_tag(tag["id"])
        
        # 按使用频率排序
        scenes.sort(key=lambda s: s["usage_count"], reverse=True)
        
        sections.append({
            "tag": {
                "id": tag["id"],
                "name": tag["name"],
                "icon": tag["icon"]
            },
            "scenes": scenes
        })
    
    # 4. 返回结果
    return {
        "quick_access": quick_access,
        "sections": sections
    }
```

## 七、总结

### 一级分区设计的核心

| 维度 | 设计 |
|------|------|
| **展示方式** | 直接按功能标签分区展示 |
| **分区来源** | 功能标签维度 |
| **场景归属** | 场景的主标签决定分区 |
| **交互方式** | 点击场景卡片代入场景 |
| **筛选器** | 不需要 |
| **Tab切换** | 不需要 |

### 优势

- ✅ 极简设计，用户一目了然
- ✅ 不需要操作筛选器
- ✅ 不需要理解复杂逻辑
- ✅ 符合用户直觉
- ✅ 移动端友好

### 适用条件

- ✅ 场景数量 20-50个
- ✅ 功能标签清晰（3-5个）
- ✅ 每个分区场景数量均衡
