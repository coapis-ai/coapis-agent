> **目标**：深入分析标签的管理、展示、使用等全方位需求

## 一、标签的本质

### 1.1 标签是什么？

**标签是多维度的场景组织方式**

```
场景是核心实体
    ↓
标签是场景的属性（可以有多个）
    ↓
标签维度是标签的分类方式
```

**举例**：

```
【会议纪要】场景
├── 功能标签：办公（用于功能分类展示）
├── 行业标签：通用（用于行业过滤）
├── 场景类型标签：日常（用于场景推荐）
├── 使用频率标签：高频（用于快捷访问）
└── 技能标签：音频转写、内容提取（用于技能触发）

【政策解读】场景
├── 功能标签：文档处理
├── 行业标签：政府
├── 场景类型标签：专业
└── 技能标签：文档解析、内容提取

【数据分析】场景
├── 功能标签：数据分析
├── 行业标签：通用
├── 场景类型标签：高级
└── 技能标签：数据处理、图表生成
```

---

### 1.2 标签的作用

| 维度 | 作用 | 示例 |
|------|------|------|
| **分类展示** | 工作台按标签分组展示场景 | 按"功能"标签分组：办公、数据分析、文档处理 |
| **筛选过滤** | 用户筛选感兴趣的场景 | 勾选"政府"标签，只显示政府相关场景 |
| **智能推荐** | 根据用户输入关键词推荐场景 | 用户输入"会议"，推荐带"会议"关键词的场景 |
| **技能触发** | 标签关联技能，场景代入时自动加载技能 | "音频处理"标签关联"音频转写"技能 |
| **权限控制** | 不同角色看到不同标签的场景 | 普通用户看不到"高级"标签的场景 |
| **使用统计** | 统计标签使用频率，优化推荐 | "办公"标签使用最多，放在前面 |

## 二、标签的管理需求

### 2.1 标签的生命周期

```
创建标签
    ↓
配置标签属性（名称、图标、关键词、技能等）
    ↓
关联场景（给场景打标签）
    ↓
启用标签（控制系统显示）
    ↓
使用和统计
    ↓
优化和调整
```

## 四、标签的使用需求

### 4.1 场景搜索和推荐

**场景1：关键词搜索**

```
用户输入：会议

系统匹配：
1. 搜索场景名称包含"会议"
2. 搜索场景描述包含"会议"
3. 搜索标签关键词包含"会议"

结果：
- 会议纪要（名称匹配 + "办公"标签包含"会议"关键词）
- 会议组织（名称匹配）
- 项目会议（描述匹配）
```

---

**场景2：智能推荐**

```
用户输入：我需要写一份会议总结

系统分析：
1. 提取关键词：会议、总结
2. 匹配标签关键词："办公"标签包含"会议"
3. 匹配场景关键词：会议纪要

推荐结果：
💡 推荐场景：会议纪要
   "我可以帮您快速生成会议纪要..."
```

### 4.2 技能自动加载

**场景代入时自动加载相关技能**：

```
用户选择：会议纪要场景
    ↓
系统读取场景标签：办公
    ↓
读取标签关联技能：音频转写、内容提取
    ↓
自动加载这些技能到当前会话
    ↓
用户可以直接使用这些技能
```

**实现**：

```python
def inject_scene(scene_id: str):
    # 1. 获取场景
    scene = get_scene(scene_id)
    
    # 2. 获取场景所有标签
    tags = get_scene_tags(scene_id)
    
    # 3. 收集所有关联技能
    skills = set()
    for tag in tags:
        skills.update(tag.related_skills)
    
    # 4. 加载技能
    for skill_id in skills:
        load_skill(skill_id)
    
    # 5. 注入系统提示词
    system_prompt = scene.system_prompt
    if skills:
        system_prompt += f"\n\n已加载技能：{', '.join(skills)}"
    
    inject_system_prompt(system_prompt)
```

### 4.3 权限控制

**基于标签的权限控制**：

```
角色：普通用户
    ↓
允许的标签：办公、通用、日常
    ↓
只能看到这些标签的场景

角色：高级用户
    ↓
允许的标签：所有标签
    ↓
可以看到所有场景
```

**实现**：

```python
def list_scenes(user_role: str, filters: dict = None):
    # 1. 获取用户允许的标签
    allowed_tags = get_role_allowed_tags(user_role)
    
    # 2. 查询场景
    scenes = query_all_scenes()
    
    # 3. 过滤场景
    result = []
    for scene in scenes:
        scene_tags = get_scene_tags(scene.id)
        
        # 检查场景的所有标签是否都在允许列表中
        if all(tag.id in allowed_tags for tag in scene_tags):
            result.append(scene)
    
    return result
```

### 4.4 使用统计

**统计标签使用频率**：

```
统计维度：
1. 标签被点击次数（筛选器中）
2. 标签关联场景被使用次数
3. 标签关键词匹配次数

用途：
1. 优化标签排序（使用多的标签排在前面）
2. 优化快捷推荐（使用多的场景推荐给用户）
3. 分析用户行为（用户偏好哪些类型的场景）
```

## 七、总结

### 标签系统核心要点

| 维度 | 需求 | 设计 |
|------|------|------|
| **管理** | 增删改查、维度分类、排序启用 | 标签管理界面、维度管理 |
| **展示** | 工作台分组、筛选器、场景卡片 | 按维度分栏、筛选过滤、标签显示 |
| **使用** | 搜索推荐、技能加载、权限控制、统计 | 关键词匹配、技能关联、权限过滤、使用统计 |

### 标签数据结构

```
标签维度（tag_dimensions）
    ↓ 包含
标签（tags）
    ├── 基本信息（id、name、icon、description）
    ├── 维度归属（dimension）
    ├── 关键词（keywords）
    ├── 关联技能（related_skills）
    ├── 显示设置（sort_order、enabled、show_in_filter）
    └── 统计数据（usage_count）
```
