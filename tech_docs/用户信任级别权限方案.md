# CoApis 用户信任级别权限方案

> 版本：v2.0（简化版）  
> 日期：2026-07-21  
> 核心思想：创建用户时设置信任级别，自动获得相应权限

---

## 一、方案核心

### 1.1 设计理念

```
传统RBAC：
用户 → 角色 → 权限 → 功能
（需要配置角色、权限映射，复杂）

信任级别：
用户 → 信任级别 → 功能
（一个字段搞定，简单直接）
```

### 1.2 信任级别定义

| 信任级别 | 名称 | 可访问功能 | 典型用户 |
|---------|------|-----------|---------|
| **Level 0** | 普通用户 | 基础功能（场景、对话、文件、设置） | 新员工、普通员工 |
| **Level 1** | 高级用户 | 基础 + 高级功能（智能体、定时任务、渠道） | 技术人员、高级员工 |
| **Level 2** | 技术用户 | 高级 + 技术功能（运行配置、心跳、MCP） | 技术专家、运维人员 |
| **Level 3** | 管理员 | 所有功能 | IT管理员、系统管理员 |

## 三、数据模型

### 3.1 用户表扩展

```python
# server/coapis/models/user.py

from enum import IntEnum

class TrustLevel(IntEnum):
    """用户信任级别"""
    NORMAL = 0      # 普通用户
    ADVANCED = 1    # 高级用户
    TECHNICAL = 2   # 技术用户
    ADMIN = 3       # 管理员

class User(BaseModel):
    id: str
    username: str
    email: Optional[str]
    
    # 核心字段：信任级别
    trust_level: TrustLevel = TrustLevel.NORMAL
    
    # 其他字段
    is_active: bool = True
    created_at: datetime
    updated_at: datetime
    
    @property
    def role(self) -> str:
        """兼容性：根据信任级别返回角色"""
        if self.trust_level >= TrustLevel.ADMIN:
            return "admin"
        return "user"
```

### 3.2 数据库迁移

```sql
-- 添加信任级别字段
ALTER TABLE users ADD COLUMN trust_level INTEGER DEFAULT 0;

-- 迁移现有用户（根据role字段）
UPDATE users SET trust_level = 3 WHERE role = 'admin';
UPDATE users SET trust_level = 0 WHERE role = 'user' OR role IS NULL;

-- 创建索引
CREATE INDEX idx_users_trust_level ON users(trust_level);
```

## 五、后端实现

### 5.1 权限检查装饰器

```python
# server/coapis/permissions/decorators.py

from functools import wraps
from fastapi import HTTPException, Depends
from ..models.user import User, TrustLevel

def require_trust_level(min_level: TrustLevel):
    """信任级别检查装饰器"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, user: User = None, **kwargs):
            if not user:
                raise HTTPException(401, "Unauthorized")
            
            if user.trust_level < min_level:
                raise HTTPException(
                    403, 
                    f"需要信任级别 {min_level.name} 或更高"
                )
            
            return await func(*args, user=user, **kwargs)
        return wrapper
    return decorator

# 使用示例
@router.post("/agents")
@require_trust_level(TrustLevel.ADVANCED)
async def create_agent(
    agent_data: AgentCreate,
    user: User = Depends(get_current_user)
):
    """创建智能体（需要高级用户）"""
    pass

@router.get("/runtime/config")
@require_trust_level(TrustLevel.TECHNICAL)
async def get_runtime_config(
    user: User = Depends(get_current_user)
):
    """查看运行配置（需要技术用户）"""
    pass
```

### 5.2 用户管理API

```python
# server/coapis/app/routers/users.py

from fastapi import APIRouter, Depends
from ..models.user import User, TrustLevel
from ..auth import get_current_user

router = APIRouter()

@router.post("/users")
@require_trust_level(TrustLevel.ADMIN)
async def create_user(
    user_data: UserCreate,
    current_user: User = Depends(get_current_user)
):
    """创建用户（仅管理员）"""
    # 验证信任级别不能超过当前用户
    if user_data.trust_level > current_user.trust_level:
        raise HTTPException(
            403, 
            "不能创建比当前用户更高级别的用户"
        )
    
    # 创建用户
    new_user = User(
        id=generate_user_id(),
        username=user_data.username,
        email=user_data.email,
        trust_level=user_data.trust_level,
        is_active=True
    )
    
    await db.users.insert_one(new_user.dict())
    
    return {"id": new_user.id, "trust_level": new_user.trust_level}

@router.put("/users/{user_id}/trust-level")
@require_trust_level(TrustLevel.ADMIN)
async def update_trust_level(
    user_id: str,
    trust_level: TrustLevel,
    current_user: User = Depends(get_current_user)
):
    """调整用户信任级别（仅管理员）"""
    # 验证信任级别不能超过当前用户
    if trust_level > current_user.trust_level:
        raise HTTPException(403, "不能设置比自己更高的信任级别")
    
    # 更新用户信任级别
    await db.users.update_one(
        {"id": user_id},
        {"$set": {"trust_level": trust_level, "updated_at": datetime.now()}}
    )
    
    # 记录审计日志
    await audit_log(
        action="update_trust_level",
        user_id=current_user.id,
        target_user_id=user_id,
        old_value=old_trust_level,
        new_value=trust_level
    )
    
    return {"success": True}
```
