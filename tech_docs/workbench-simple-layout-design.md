# 工作台左右分栏设计（极简版）

> **核心设计**：左侧菜单 + 右侧内容  
> **极简原则**：不添加任何分组、不添加任何额外功能  
> **创建日期**：2026-07-17

---

## 二、左侧菜单

### 2.1 菜单项

```
左侧菜单
├── 📁 办公
├── 📊 数据分析
├── 📄 文档处理
└── 💬 沟通协作
```

**没有分组标题！没有"业务场景"、"快速访问"这些！**

---

### 2.2 菜单项设计

```
┌────────────┐
│ 📁 办公 ✓  │ ← 图标 + 名称 + 选中标记
└────────────┘
```

**交互**：
- 点击 → 右侧切换显示对应场景
- 当前选中项高亮

---

### 2.3 菜单扩展

左侧菜单分为两类：

**场景菜单**（展示场景卡片）：
```
左侧菜单
├── 📁 办公
├── 📊 数据分析
├── 📄 文档处理
└── 💬 沟通协作
```

**管理菜单**（跳转管理页面）：
```
左侧菜单
└── ⚙️ 设置
    ├── 场景管理
    ├── 技能管理
    ├── 提供商管理
    └── ...
```

**说明**：
- 场景菜单：点击后在右侧显示场景卡片网格
- 管理菜单：点击后跳转到对应的管理页面（或展开子菜单）

## 六、API设计

### 6.1 获取菜单

```python
GET /api/workbench/menu

响应：
{
  "items": [
    {
      "id": "office",
      "name": "办公",
      "icon": "📁"
    }
  ]
}
```

---

```python
GET /api/workbench/scenes?menu_id=office
```

## 七、总结

### 极简设计的核心

| 维度 | 设计 |
|------|------|
| **左侧菜单** | 只有菜单项，没有分组 |
| **右侧内容** | 只有场景卡片，没有工具栏 |
| **交互** | 点击菜单项 → 右侧切换场景 |

### 为什么极简？

- ✅ 用户只想要简单的分类和场景
- ✅ 不需要分组标题、收藏、最近使用等额外功能
- ✅ 未来需要时再添加，不提前设计
