# 工作台一级结构设计分析

> **核心问题**：如何用一级结构实现场景分类和查找  
> **目标**：简单、直接、高效  
> **创建日期**：2026-07-17

---

## 一、为什么一级就够了？

### 1.1 用户视角

**用户的真实需求**：
1. 快速找到想要的场景
2. 看到所有可用的场景
3. 不需要理解复杂的层级

**一级结构的优势**：
- ✅ 所有场景平铺展示，一目了然
- ✅ 配合筛选和搜索，快速定位
- ✅ 不需要点击展开，减少操作
- ✅ 符合"扁平化"设计趋势

---

### 1.2 场景数量的现实

**初期场景数量**：20-30个

| 场景类型 | 数量 |
|---------|------|
| 通用办公 | 10个 |
| 数据分析 | 5个 |
| 文档处理 | 5个 |
| 行业场景 | 5-10个 |

**20-30个场景，一级列表完全够用！**

**对比**：
- 手机App列表：几十上百个应用，一级列表
- 商品列表：成百上千商品，一级列表 + 筛选
- 文件列表：几十上百文件，一级列表 + 搜索

**结论**：场景数量不多，一级列表 + 筛选/搜索 足够

---

## 二、一级结构设计方案

### 方案：一级扁平列表 + 标签筛选

```
工作台
├── 快捷功能区（横向滚动）
│   └── [会议纪要] [工作报告] [政策解读] ...
│
├── 筛选区
│   ├── 功能标签：[全部] [办公] [数据分析] [文档处理]
│   ├── 行业标签：[全部] [通用] [政府] [金融]
│   └── 搜索框：[搜索场景...]
│
└── 场景列表（扁平卡片列表）
    ├── [会议纪要]  📝  办公 · 通用 · 高频
    ├── [工作报告]  📊  办公 · 通用
    ├── [政策解读]  📋  文档处理 · 政府
    ├── [数据分析]  📈  数据分析 · 通用
    └── ...
```

**核心思想**：
- **没有分组标题**，所有场景平铺
- **标签在场景卡片上显示**，用户一目了然
- **通过筛选器过滤**，快速找到目标场景

### 3.2 快捷功能区

**设计要点**：
- 横向滚动卡片，最多显示8-10个
- 显示图标 + 名称
- 点击直接代入场景

**数据来源**：
- 用户高频使用的场景（系统统计）
- 用户手动固定的场景
- 管理员推荐的场景

**交互**：
- 点击 → 代入场景
- 长按/右键 → 固定/取消固定
- 拖拽排序（可选）

---

### 3.3 筛选区

**功能标签**：
- 横向单选按钮组
- 默认选中"全部"
- 选中状态高亮
- 点击切换

**行业标签**：
- 横向单选按钮组
- 默认选中"全部"
- 可以和功能标签组合筛选

**搜索框**：
- 输入关键词实时搜索
- 搜索范围：场景名称、描述、标签关键词
- 清空按钮

**组合逻辑**：
```
筛选结果 = 功能标签筛选 AND 行业标签筛选 AND 搜索关键词
```

## 四、筛选逻辑详解

### 4.1 单维筛选

**场景1：只选功能标签**

```
用户点击：[办公]

显示：所有标签包含"办公"的场景
- 会议纪要（办公 · 通用）
- 工作报告（办公 · 通用）
- 邮件撰写（办公 · 通用）
- 通知公告（办公 · 通用）
```

---

### 4.2 多维组合筛选

**场景2：功能 + 行业**

```
用户点击：
  功能：[办公] ✓
  行业：[政府] ✓

显示：标签同时包含"办公"和"政府"的场景
- 公文起草（办公 · 政府）
- 项目申报（办公 · 政府）

注意：不显示"会议纪要"（虽然是办公，但不是政府）
```

---

### 4.3 搜索 + 筛选

**场景3：搜索 + 标签筛选**

```
用户操作：
  功能：[办公] ✓
  搜索："会议"

显示：标签包含"办公"且名称/描述/关键词包含"会议"的场景
- 会议纪要（办公 · 通用）
- 会议组织（办公 · 政府）
```

## 五、标签在场景卡片上的显示

### 5.1 显示策略

**原则**：
- 显示最重要的2-3个标签
- 标签按维度优先级排序
- 不同维度用不同颜色区分

**示例**：

```
标签：办公 · 通用 · 高频
      (功能)  (行业) (频率)

标签：文档处理 · 政府 · 专业
      (功能)    (行业) (类型)
```

---

### 5.2 颜色方案

**按维度区分颜色**：

| 维度 | 颜色 | 示例 |
|------|------|------|
| 功能标签 | 蓝色 | 办公、数据分析 |
| 行业标签 | 紫色 | 通用、政府、金融 |
| 场景类型 | 绿色 | 日常、专业、高级 |
| 使用频率 | 橙色 | 高频、常用、一般 |

**实现**：

```css
.tag-function { color: #1890ff; }
.tag-industry { color: #722ed1; }
.tag-type { color: #52c41a; }
.tag-frequency { color: #faad14; }
```

### 7.2 筛选逻辑实现

```python
def list_scenes(
    function_tag: Optional[str] = None,
    industry_tag: Optional[str] = None,
    search_keyword: Optional[str] = None,
    page: int = 1,
    page_size: int = 20
) -> dict:
    """获取场景列表（支持筛选和搜索）"""
    
    # 1. 查询所有启用的场景
    scenes = query_all_enabled_scenes()
    
    # 2. 功能标签筛选
    if function_tag:
        scenes = [
            s for s in scenes
            if function_tag in s.tags
        ]
    
    # 3. 行业标签筛选
    if industry_tag:
        scenes = [
            s for s in scenes
            if industry_tag in s.tags
        ]
    
    # 4. 关键词搜索
    if search_keyword:
        scenes = [
            s for s in scenes
            if search_keyword in s.name
            or search_keyword in s.description
            or any(
                search_keyword in tag.keywords
                for tag in get_tags(s.tags)
            )
        ]
    
    # 5. 排序（按使用频率）
    scenes.sort(key=lambda s: s.usage_count, reverse=True)
    
    # 6. 分页
    total = len(scenes)
    start = (page - 1) * page_size
    end = start + page_size
    scenes = scenes[start:end]
    
    # 7. 返回结果
    return {
        "total": total,
        "page": page,
        "page_size": page_size,
        "scenes": scenes
    }
```

## 八、总结

### 一级结构的优势

| 维度 | 优势 |
|------|------|
| **简单** | 扁平列表，没有层级，用户一目了然 |
| **直接** | 所有场景平铺，不需要点击展开 |
| **高效** | 筛选 + 搜索，快速定位场景 |
| **灵活** | 标签多维度组合，满足不同需求 |

### 核心设计要点

1. **快捷功能区**：高频场景快速访问
2. **筛选区**：标签筛选 + 搜索，精准定位
3. **场景列表**：扁平列表，标签直接显示在卡片上
4. **交互简单**：点击卡片代入场景，点击标签快速筛选

### 适用条件

- ✅ 场景数量 20-50个
- ✅ 用户需要快速找到场景
- ✅ 标签系统完善
- ✅ 筛选和搜索功能强大
