## 一、用户需求分析

### 1.1 用户提出的6个核心想法

| # | 想法 | 分析 | 建议 |
|---|------|------|------|
| 1 | **首页看板** | 综合性Dashboard，未来可扩展 | ✅ 核心设计，需规划 |
| 2 | **办公** | 替代"场景"，更通俗易懂 | ✅ 命名优化 |
| 3 | **我的领域** | 根据岗位标签推荐，替代对话菜单 | ✅ 创新设计，很有价值 |
| 4 | **我的空间** | 替代"文件"，更有归属感 | ✅ 命名优化 |
| 5 | **知识库** | 企业版功能 | ✅ 企业版专属 |
| 6 | **设置→个人管理** | 集中管理个人资源 | ✅ 实现方案 |

### 1.2 关键洞察

```
用户的核心思考：
1. 菜单应该贴近用户视角（办公、我的空间、我的领域）
2. 对话不需要单独菜单（浮动图标更灵活）
3. 首页应该是一个综合性的Dashboard
4. "我的领域"概念：根据专业标签个性化推荐
5. 企业版和社区版功能差异（知识库）
```

## 二、菜单结构设计

### 2.1 社区版菜单

```
工作台菜单（5项）：
├─ 首页          ← 综合看板（未来扩展）
├─ 办公          ← 办公类场景（替代"场景"）
├─ 我的领域      ← 专业标签 + 个性化推荐（替代"对话"）
├─ 我的空间      ← 文件 + 个人资源（替代"文件"）
└─ 设置          ← 个人管理中心
```

### 2.2 企业版菜单

```
工作台菜单（6项）：
├─ 首页          ← 综合看板（未来扩展）
├─ 办公          ← 办公类场景
├─ 我的领域      ← 专业标签 + 个性化推荐
├─ 我的空间      ← 文件 + 个人资源
├─ 知识库        ← 企业知识管理（企业版专属）
└─ 设置          ← 个人管理中心
```

### 2.3 对话功能调整

```
移除对话菜单，改用浮动图标：

└─ 优势：随时随地可用，不占用菜单
```

## 四、对话功能调整

### 4.1 移除对话菜单

```
原因：
1. 对话入口太多（首页、场景卡片、浮动图标等）
2. 浮动图标更灵活，随时随地可用
3. 菜单项更聚焦核心业务功能
```

```
设计：

└─ 状态：
    ├─ 展开状态：聊天窗口

实现效果：
```
