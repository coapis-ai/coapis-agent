# CoApis 个人管理中心设计方案

> 版本：v3.0（极简版）  
> 日期：2026-07-21  
> 核心思想：菜单极简 + 个人管理中心 + 能力导向

---

## 一、核心理念

### 1.1 问题分析

```
之前方案的问题：
├─ Level 0/1/2 分级太细，必要性不大
├─ 菜单项太多，即使分级也显得复杂
└─ 权限限制过于复杂，不够灵活

用户真正需要的：
├─ 菜单极简（5个基础菜单）
├─ 个人功能集中在一个入口（个人管理中心）
├─ 能力足够就能使用，而不是权限限制
└─ 通过聊天或界面自然管理
```

### 1.2 设计思想

```
传统RBAC：限制导向
用户 → 角色 → 权限 → 功能
（"你没有权限"）

新方案：能力导向
用户 → 能力 → 功能
（"你只要会用，就能用"）

核心区别：
- 传统：先判断权限，无权限就拒绝
- 新方案：提供入口，用户能力足够就能使用
```

### 1.3 权限简化

```
只需要两个级别：
├─ 普通用户：个人资源管理（智能体、定时任务、渠道等）
└─ 管理员：系统资源管理 + 用户管理

关键：
- 不再细分Level 0/1/2
- 个人资源：用户只要会用，就能管理
- 系统资源：只有管理员能管理
```

## 二、菜单极简化

### 2.1 工作台菜单（所有用户）

```
工作台菜单（5项）：
├─ 首页          ← 推荐场景、最近使用、快速入口
├─ 场景          ← 浏览和使用场景

├─ 文件          ← 个人文件
└─ 设置          ← 个人管理中心入口 ⭐
```

**关键改进**：
- ✅ 菜单只有5项，极简
- ✅ 所有个人功能都在"设置"里
- ✅ 不需要在菜单中显示智能体、定时任务等

### 2.2 控制台菜单（仅管理员）

```
控制台菜单（管理员专属）：
├─ 概览          ← 系统统计
├─ 场景管理      ← 场景CRUD
├─ 模型管理      ← 模型配置
├─ 用户管理      ← 用户CRUD
├─ 系统监控      ← 运行状态、心跳、MCP
├─ 审计日志      ← 操作记录
└─ 系统设置      ← 全局配置
```

## 四、能力导向权限设计

### 4.1 核心概念

```
权限分类：
├─ 个人资源（能力导向）
│   ├─ 用户只要有能力，就能管理
│   ├─ 不需要复杂权限配置
│   └─ 通过界面或聊天自然使用
│
└─ 系统资源（角色导向）
    ├─ 只有管理员能管理
    └─ 影响全局，需要权限控制
```

### 4.2 个人资源管理

| 资源类型 | 管理方式 | 能力要求 |
|---------|---------|---------|
| **个人智能体** | 个人管理中心 / 聊天创建 | 理解智能体概念 |
| **个人定时任务** | 个人管理中心 / 聊天创建 | 理解定时任务概念 |
| **个人渠道** | 个人管理中心 / 聊天配置 | 有企业微信/飞书账号 |
| **个人文件** | 文件管理页面 | 基础文件操作能力 |
| **个人对话** | 对话页面 | 基础对话能力 |

**关键思想**：
- ✅ 不限制权限，用户只要有能力就能用
- ✅ 通过界面引导，降低使用门槛
- ✅ 通过聊天辅助，帮助用户理解和使用

### 4.3 系统资源管理（管理员专属）

| 资源类型 | 管理方式 | 权限要求 |
|---------|---------|---------|
| **场景管理** | 控制台 | 管理员 |
| **模型配置** | 控制台 | 管理员 |
| **用户管理** | 控制台 | 管理员 |
| **系统监控** | 控制台 | 管理员 |
| **审计日志** | 控制台 | 管理员 |
| **系统设置** | 控制台 | 管理员 |

## 五、聊天管理增强

### 5.1 通过聊天管理个人资源

```
用户聊天 → AI理解意图 → 执行操作 → 确认结果

示例对话：
用户："帮我创建一个每天早上9点发日报的定时任务"
AI：  "好的，我已经为您创建了定时任务：
       - 任务名称：日报发送
       - 执行时间：每天 09:00
       - 是否启用？
       [确认] [取消]"

用户："确认"
AI：  "✅ 定时任务已创建并启用，您可以在个人管理中心查看"
```

### 5.2 AI辅助降低门槛

```
AI能力：
├─ 意图识别：理解用户想做什么
├─ 引导操作：不知道如何操作时，引导用户
├─ 参数提取：从对话中提取必要参数
├─ 风险提示：提醒用户操作的影响
└─ 执行确认：重要操作需要确认

示例：
用户："我想配置企业微信"
AI：  "好的，配置企业微信需要以下信息：
       1. 企业微信CorpID
       2. AgentID
       3. Secret
       
       请提供这些信息，或者点击[这里]进入配置页面"
```

## 八、后端实现

### 8.1 个人资源API

```python
# server/coapis/app/routers/agents.py

@router.get("/agents")
async def list_agents(
    created_by: Optional[str] = None,
    is_public: bool = False,
    user: User = Depends(get_current_user)
):
    """获取智能体列表"""
    query = {}
    
    if created_by:
        # 查询特定用户的智能体
        query["created_by"] = created_by
    elif user.role != UserRole.ADMIN:
        # 普通用户只能看到自己的 + 公开的
        query["$or"] = [
            {"created_by": user.id},
            {"is_public": True}
        ]
    
    return await db.agents.find(query).to_list(None)

@router.post("/agents")
async def create_agent(
    agent_data: AgentCreate,
    user: User = Depends(get_current_user)
):
    """创建智能体（任何用户都可以）"""
    agent = Agent(
        id=generate_id(),
        name=agent_data.name,
        description=agent_data.description,
        created_by=user.id,  # 自动设置创建者
        is_public=False
    )
    
    await db.agents.insert_one(agent.dict())
    return agent

@router.put("/agents/{agent_id}")
async def update_agent(
    agent_id: str,
    agent_data: AgentUpdate,
    user: User = Depends(get_current_user)
):
    """更新智能体"""
    agent = await db.agents.find_one({"id": agent_id})
    
    if not agent:
        raise HTTPException(404, "Agent not found")
    
    # 权限检查：只能修改自己创建的
    if agent["created_by"] != user.id and user.role != UserRole.ADMIN:
        raise HTTPException(403, "No permission")
    
    # 更新
    await db.agents.update_one(
        {"id": agent_id},
        {"$set": agent_data.dict(exclude_unset=True)}
    )
    
    return {"success": True}
```

### 8.2 简化权限检查

```python
# server/coapis/permissions/decorators.py

def require_admin(func):
    """管理员权限检查"""
    @wraps(func)
    async def wrapper(*args, user: User = None, **kwargs):
        if not user:
            raise HTTPException(401, "Unauthorized")
        if user.role != UserRole.ADMIN:
            raise HTTPException(403, "Admin required")
        return await func(*args, user=user, **kwargs)
    return wrapper

def require_owner(resource_id: str, resource_type: str):
    """资源所有权检查"""
    async def decorator(func):
        @wraps(func)
        async def wrapper(*args, user: User = None, **kwargs):
            if not user:
                raise HTTPException(401, "Unauthorized")
            
            # 获取资源
            resource = await get_resource(resource_type, resource_id)
            
            # 检查所有权
            if resource.created_by != user.id and user.role != UserRole.ADMIN:
                raise HTTPException(403, "No permission")
            
            return await func(*args, user=user, **kwargs)
        return wrapper
    return decorator
```
