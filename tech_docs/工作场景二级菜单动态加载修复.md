## 🚨 问题描述

### 错误实现

**工作场景二级菜单**：
- ❌ 硬编码了9个业务领域
- ❌ 与标签管理系统完全脱节
- ❌ 点击分类后无法正确过滤场景

**用户反馈**：
> "工作场景下的二级菜单，数据来自哪儿的？你自己编的吗？这里既然是场景，怎么能脱离开'标签管理'？应该按照标签管理的分类来呀，怎么能独立搞一个？而且每个分类应该要显示场景管理的数据呀"

## ✅ 正确设计

### 数据来源

```
工作场景二级菜单
  ├── 数据来源：标签管理系统
  ├── 使用：type='category' 且 parent_id='domain' 的标签
  └── 过滤：场景的 primary_tag_id 或 tag_ids 匹配分类ID
```

## 🛠️ 修复方案

### 1. 前端修改

**menuConfig.tsx**：
- 删除硬编码的 `WORKBENCH_SUB_MENUS`
- 二级菜单改为动态加载

**Sidebar.tsx**：
```typescript
// 添加状态
const [workbenchCategories, setWorkbenchCategories] = useState<any[]>([]);

// 加载分类数据
useEffect(() => {
  fetch('/api/scenes/categories/grouped')
    .then(res => res.json())
    .then(data => {
      const domainCategories = data?.dimensions?.domain?.categories || [];
      setWorkbenchCategories(domainCategories);
    })
    .catch(err => {
      console.error('Failed to load workbench categories:', err);
    });
}, []);

// 动态生成二级菜单
const collapsedNavItems = MAIN_MENU_ITEMS.map(item => {
  const menuItem: any = { ... };
  
  // 为工作场景添加动态二级菜单
  if (item.key === 'workbench' && workbenchCategories.length > 0) {
    menuItem.children = workbenchCategories.map((cat: any) => ({
      key: cat.id,
      icon: <span>{cat.icon}</span>,
      path: `/workbench/${cat.id}`,
      label: cat.name,
    }));
  }
  
  return menuItem;
});
```

**Workbench/index.tsx**：
```typescript
// 场景过滤逻辑（使用primary_tag_id或tag_ids匹配）
const filteredScenes = useMemo(() => {
  return scenes.filter(scene => {
    if (scene.status !== 'active') return false;
    
    // Filter by category from URL
    if (categoryParam !== 'all') {
      const matchPrimaryTag = scene.primary_tag_id === categoryParam;
      const matchTagIds = scene.tag_ids && scene.tag_ids.includes(categoryParam);
      if (!matchPrimaryTag && !matchTagIds) {
        return false;
      }
    }
    
    return true;
  });
}, [scenes, categoryParam, searchText, selectedTag]);
```

### 场景过滤

- 点击"办公协作" → 无法过滤场景（分类ID与场景category字段不匹配）

- 使用`nature`维度的分类（通用分类）
- 场景的`primary_tag_id`匹配nature维度的分类ID
- domain维度（按领域分类）暂不作为二级菜单

## 🎯 架构原则

### 数据一致性

✅ **正确做法**：
- 所有分类数据来自标签管理系统
- 场景通过 `primary_tag_id` 和 `tag_ids` 关联标签
- 前端动态加载，自动同步

❌ **错误做法**：
- 前端硬编码分类
- 与标签管理系统脱节
- 数据不一致
