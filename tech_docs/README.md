## 📝 文档说明

### 设计文档 (design/)

#### 1. 工作空间整合设计
**文件**: `workspace-integration-design.md`

**内容**：
- 在聊天界面中整合工作空间和知识库的两种方案
- 方案A：聊天界面整合（推荐）
- 方案B：独立工作台
- 技术实现、性能优化、移动端适配

**状态**: ✅ 已完成

---

#### 2. 卡片化工作台设计
**文件**: `card-based-workbench-design.md`

**内容**：
- 从"聊天工具"到"业务工作台"的理念转变
- 卡片化设计：聊天、文件、图表、知识库、任务、笔记
- 多角色支持：领导、研发、运营
- 系统集成：ERP、CRM、BI
- AI增强：智能推荐、自动工作流

**状态**: ✅ 已完成

---

#### 3. 嵌入式AI助手设计
**文件**: `embedded-ai-assistant-design.md`

**内容**：
- CoApis嵌入其他系统（OA/ERP/CRM）的方案
- 四种嵌入方式：浮动按钮、侧边栏、嵌入式面板、API集成
- 深度集成：上下文感知、双向数据交互、权限控制
- SDK设计、实施路线图

**状态**: ✅ 已完成

## 📝 文档编写规范

### 命名规范

- 使用小写字母和连字符：`feature-name-design.md`
- 设计文档以 `-design.md` 结尾
- 架构文档以 `-architecture.md` 结尾
- 指南文档以 `-guide.md` 结尾

### 文档结构

```markdown
# 标题

## 📋 背景
## 🎯 目标
## 💡 方案
## 🔧 实现
## 📊 对比
## ✅ 总结
```

### 更新记录

每个文档应在末尾包含更新记录：

```markdown
## 📝 更新记录

- 2026-07-14: 初版完成
- 2026-07-15: 补充技术细节
```
