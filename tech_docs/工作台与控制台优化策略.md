# CoApis 工作台与控制台优化策略

> 分析日期：2026-07-21  
> 核心思想：**工作台面向普通用户，控制台面向管理员**

---

## 一、当前架构分析

### 1.1 路由结构

```
当前路由结构：
/ (根路径)
├─ /chat                    ← 默认首页（对话）
├─ /workbench               ← 工作台（独立页面）
│
├─ /channels                ← 控制组
├─ /sessions
├─ /cron-jobs
├─ /heartbeat
│
├─ /myspace                 ← 智能体组
├─ /skills
├─ /tools
├─ /mcp
├─ /agent-config
├─ /agent-stats
│
├─ /agents                  ← 设置组
├─ /models
├─ /security
├─ /token-usage
├─ /backups
├─ /voice-transcription
├─ /evolution
└─ /admin                   ← 后台管理
```

### 1.2 问题识别

#### ⚠️ 问题1：用户角色混乱
- 普通用户和管理员看到相同的菜单
- 管理功能（models、agents、admin）对普通用户可见
- 用户容易误操作

#### ⚠️ 问题2：工作台定位不清
- 工作台是独立页面，但不是默认首页
- 工作台和控制台平级，用户需要在两个入口切换
- 场景化功能（工作台）和通用对话（/chat）重复

#### ⚠️ 问题3：菜单层级过多
- 控制组、智能体组、设置组都是二级菜单
- 用户需要点击2次才能访问功能
- 高频功能（会话、文件）埋在二级菜单

### 策略2：统一工作台模式

**核心理念**：工作台作为唯一入口，控制台功能嵌入

```
统一工作台：
├─ 🏠 首页（场景化入口）
├─ 💬 对话（通用对话）
├─ 📁 文件
├─ 🔮 技能
├─ ⚙️ 设置（普通用户：个人设置 / 管理员：完整设置）
└─ 👑 管理（仅管理员可见）
```

**优点**：
- 单一入口，用户不会迷路
- 角色权限控制功能可见性

**缺点**：
- 管理员需要在"设置"和"管理"之间切换
- 不如双入口模式清晰

---

### 策略3：混合模式

**核心理念**：工作台和控制台共享部分功能

```
混合模式：
工作台（用户 + 管理员共享）：
├─ 场景列表
├─ 我的对话
├─ 我的文件
└─ 个人设置

控制台（管理员专属）：
├─ 对话（通用）
├─ 场景管理
├─ 系统配置
├─ 用户管理
└─ 数据分析
```

**优点**：
- 灵活性高
- 管理员也能使用工作台

**缺点**：
- 两个入口可能让用户困惑
- 需要明确切换机制

## 三、推荐方案：双入口模式

### 3.1 用户旅程设计

#### 普通用户旅程
```
1. 登录 → 自动跳转到工作台
2. 选择场景 → 开始对话
3. 管理文件 → 上传/下载
4. 定制技能 → 个人优化
5. 查看设置 → 个人偏好
```

#### 管理员旅程
```
1. 登录 → 可选择工作台或控制台
2. 工作台：测试场景、日常使用
3. 控制台：配置系统、管理用户
4. 数据分析：监控使用情况
5. 系统优化：调整配置
```

### 3.2 实施步骤

#### 第一阶段：UI分离（1周）

## 六、技术实施方案

### 6.1 前端路由重构

```typescript
// 新的路由结构
const routes = [
  // 工作台入口
  {
    path: '/workbench',
    component: WorkbenchLayout,
    children: [
      { path: '', component: WorkbenchHome },
      { path: 'scenes', component: SceneList },
      { path: 'chat/:sceneId', component: SceneChat },
      { path: 'files', component: MyFiles },
      { path: 'settings', component: PersonalSettings },
    ],
    // 权限：所有用户
    permission: ['user', 'admin'],
  },
  
  // 控制台入口
  {
    path: '/console',
    component: ConsoleLayout,
    children: [
      { path: '', component: Dashboard },
      { path: 'chat', component: Chat },
      { path: 'scenes', component: SceneManagement },
      { path: 'agents', component: AgentManagement },
      { path: 'models', component: ModelManagement },
      { path: 'users', component: UserManagement },
      { path: 'analytics', component: Analytics },
    ],
    // 权限：管理员完全访问，普通用户只读
    permission: ['admin'], // 完全权限
    readOnly: ['user'],    // 只读权限
  },
  
  // 默认跳转
  {
    path: '/',
    redirect: (user) => user.role === 'admin' ? '/console' : '/workbench',
  },
];
```

### 6.2 权限控制中间件

```typescript
// 路由守卫
function AuthGuard({ user, route }) {
  // 工作台：所有用户可访问
  if (route.path.startsWith('/workbench')) {
    return true;
  }
  
  // 控制台：管理员完全访问，普通用户只读
  if (route.path.startsWith('/console')) {
    if (user.role === 'admin') {
      return true;
    }
    // 普通用户：只读模式
    return { readOnly: true };
  }
  
  return false;
}
```
