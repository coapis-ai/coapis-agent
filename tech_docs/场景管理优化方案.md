## 当前问题

### 问题1："办公"菜单
- **现状**：126个场景平铺展示，没有分组
- **用户痛点**：
  - 找场景困难（要翻很久）
  - 不知道有哪些业务领域
  - 没有筛选功能
  - 排序混乱

### 问题2："我的场景"菜单
- **现状**：
  - "已启用"Tab：用户选择的常用场景
  - "全部场景"Tab：所有系统场景（和"办公"重复！）
- **原始设计意图**：作为"我的领域"，根据用户职能职责推荐场景
- **当前问题**：
  - 和"办公"功能重复
  - 失去了"领域"的原意
  - 菜单冗余

### 问题3：场景分类混乱
- **数据现状**：
  - 场景总数：126个
  - 业务领域：9个（办公协作、数据处理、审批服务等）
  - 行业标签：17个（国土资源、环境保护、农业等）
- **问题**：
  - 两个维度混在一起（业务领域 + 职能领域）
  - 用户不知道从哪里找场景

## 方案对比

### 方案A：恢复原始设计（职能导向）

```
├── 办公 💼
│   └── 办公协作场景（会议纪要、邮件撰写、文档处理等）
│
├── 我的领域 🎯
│   ├── 选择我的职能标签（多选）
│   └── 推荐场景（基于职能标签）
```

**优点**：
- 符合原始设计意图
- "我的领域"名副其实
- 按职能推荐，更精准

**缺点**：
- 办公场景太少（只有21个办公协作场景）
- 其他业务领域场景无处安放

**结论**：❌ 不推荐

### 方案B：双入口设计（业务领域 + 职能领域）

```
├── 办公 💼
│   ├── 办公协作
│   ├── 数据处理
│   ├── 审批服务
│   ├── 监管执法
│   └── ...
│
├── 我的领域 🎯
│   ├── 选择职能标签
│   └── 推荐相关场景
```

**优点**：
- 两个入口，满足不同需求
- "办公"= 按业务领域浏览
- "我的领域"= 按职能推荐

**缺点**：
- 两个菜单功能有重叠
- 用户可能困惑"该去哪里找场景"

**结论**：❌ 不推荐

## 推荐方案

**方案E：首页 + 工作场景**

### 核心设计理念

| 菜单 | 定位 | 用户场景 | 核心功能 |
|------|------|---------|---------|
| **首页** | 个性化、快速访问 | "我常用的场景是什么？" | 推荐场景、最近使用、收藏、热门 |
| **工作场景** | 浏览发现 | "我需要什么场景？" | 按业务领域分组、搜索筛选 |
| **我的空间** | 文件管理 | "我的文件在哪里？" | 文件上传、下载、管理 |
| **设置** | 系统配置 | "我需要调整什么？" | 个人信息、系统配置、权限管理 |

---

### 为什么推荐方案E？

#### 1. 符合主流产品设计

**钉钉**：
```
├── 首页（消息、待办、推荐）
├── 工作（应用中心）
├── 通讯录
└── 我的
```

**飞书**：
```
├── 首页（消息、工作台）
├── 工作台（应用）
├── 通讯录
└── 我的
```

**企业微信**：
```
├── 消息
├── 工作台（应用）
├── 通讯录
└── 我
```

#### 2. 解决所有当前问题
- ✅ 场景不再平铺，按业务领域分组
- ✅ 删除冗余菜单，结构清晰
- ✅ 个性化推荐放在首页，更合理
- ✅ 职能标签在首页选择，逻辑顺畅

#### 3. 首页可扩展性强
不只是场景推荐，可以做更多：
- 📊 数据统计（Token使用量、场景使用次数）
- ✅ 待办事项（场景对话中的待办）
- 📰 系统公告
- 🎯 任务中心
- 🔔 消息通知

## 风险与应对

### 风险1：用户习惯改变

**应对**：
- 提供引导提示
- 首次进入首页时显示功能介绍
- 保留数据迁移，确保用户收藏不丢失

### 风险2：首页性能

**应对**：
- 使用缓存减少API调用
